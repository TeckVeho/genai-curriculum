# __第05回：Cowork入門 (3)：プロジェクト活用＋スケジュールタスク＋複合タスク体験__

---

## __1\. 講義概要__

- __フェーズ:__ 座学（実践体験）
- __所要時間:__ 60分
- __主要機能:__ Claude Desktop — Cowork（プロジェクト・スケジュールタスク・複合タスク）
- __形式:__ オンライン

### __狙い__

Cowork の**プロジェクト機能**（タスク・ファイル・指示をまとめた永続的ワークスペース）、**スケジュールタスク**（定期自動実行）、**複合タスク**（複数の処理を1回の指示で組み合わせる）を体験する。第03〜04回で学んだ個別タスク実行を「業務の仕組み」として構築する力を身につけ、第06回以降のワークショップで自分の課題に応用する準備を完了させる。

### __到達目標__

1. Cowork のプロジェクトを作成し、指示・ファイル・メモリをまとめて管理できる
2. スケジュールタスク（定期実行）の設定方法を理解し、デモを見て仕組みをつかんでいる
3. 複合タスク（読み込み → 加工 → 文書生成 を1回の指示で実行）を体験できる
4. Cowork ソリューション設計フローチャートを理解し、自分の業務課題の解決方針をイメージできる
5. 第06回以降のワークショップで取り組む課題を具体的に決められる

### __準備物（講師）__

- プロジェクター・スクリーン（またはオンライン画面共有環境）
- 講師用PC（Claude Desktop の Cowork が動作する状態）
- 配布資料：**Cowork ソリューション設計フローチャート**（本テキスト4節参照）
- 事前に作成したデモ用プロジェクト（月次報告タスクが設定済みのもの）
- 配布用サンプルデータ：前回の `配送実績_元データ.csv`（再利用）

### __準備物（受講者）__

- PC（macOS または Windows）
- Claude Desktop アプリ（最新バージョン。Cowork の実行には Pro 以上が必要）
- 前回の棚卸しシート・第04回テイクアウェイシート（手元に置いておく）

### __【05\-20分】 プロジェクト機能：作業スペースを構築する__

- __フェーズ:__ デモ＋ハンズオン
- __スライド内容:__
	- __タイトル:__ プロジェクト ── 業務ごとの「専用デスク」を作る

- __講師スクリプト案:__  
「Cowork の**プロジェクト**は、特定の業務に関するタスク・ファイル・指示・メモリをひとまとめにした**永続的な作業スペース**です。  
毎回ゼロから説明しなくても、プロジェクトを開くだけで Claude が『この業務はこういう前提で、こういうファイルを使って、こういう手順で進める』と理解してくれます。  
物流で例えると、**業務ごとに自分専用の作業デスクがある**イメージです。デスクの上には必要な書類と手順書がいつも置いてある状態。」

- __デモ操作手順:__
	1. Cowork のサイドバーで「プロジェクト」→「新しいプロジェクトを作成」
	2. プロジェクト名：「月次配送レポート」
	3. プロジェクト指示を設定：
	```
	このプロジェクトは月次の配送実績レポート作成に使います。
	- 入力：配送実績のCSVファイル（列：日付・配送先・品名・数量・単価・金額）
	- 出力：集計Excel（配送先別集計＋グラフ）＋上司向けサマリーメール文面
	- 文体：ビジネス文体、簡潔に
	```
	4. プロジェクトにファイル（前回の CSV）を追加
	5. プロジェクト内でタスクを実行：「今月分のレポートを作成してください」
	6. プロジェクトの指示が自動的に参照されることを確認

- __ハンズオン（受講者）:__  
「では皆さんも、自分の Cowork で同じプロジェクトを作ってみてください。」
	- プロジェクト作成 → 指示設定 → ファイル追加 → タスク実行

- __解説ポイント:__
	- 「プロジェクト指示を1回設定すれば、毎回同じ前提条件を書く必要がありません」
	- 「ファイルをプロジェクトに追加しておくと、Claude が常にそのファイルを参照できます」
	- 「プロジェクトにはメモリ機能もあり、過去のタスクの学びを蓄積できます」

### __【20\-30分】 スケジュールタスク：定期実行の仕組み__

- __フェーズ:__ デモ（受講者は見学中心）
- __スライド内容:__
	- __タイトル:__ スケジュールタスク ── 「毎週月曜に○○を実行」を自動化する

- __講師スクリプト案:__  
「Cowork には**スケジュールタスク**という機能があります。『毎週月曜にこの処理を自動実行する』『毎月1日にレポートを作る』といった定期タスクを設定できます。  
ただし、PCが起動中かつ Claude Desktop が開いている間だけ実行されます。今日は設定方法のデモを見てもらいます。」

- __デモ操作手順:__
	1. Cowork のタスク入力欄に `/schedule` と入力（またはサイドバーの「Scheduled」を選択）
	2. タスク内容を設定：「月次配送レポートプロジェクトの最新データでレポートを作成してください」
	3. 実行頻度を設定：毎月1日 9:00
	4. 設定内容を確認して保存
	5. サイドバーの「Scheduled」でスケジュール一覧に追加されたことを確認

- __補足（スライドで整理）:__

| 項目 | 内容 |
| --- | --- |
| 設定方法 | `/schedule` コマンド or サイドバー |
| 実行条件 | PCが起動中 かつ Claude Desktop が開いている |
| 頻度の例 | 毎日・毎週・毎月・カスタム |
| 編集・削除 | サイドバーの「Scheduled」からいつでも変更可 |

- __業務活用イメージ:__
	- 毎週月曜：先週の配送実績の集計Excel自動生成
	- 毎月1日：月次レポートの下書き自動作成
	- 毎日朝：ダウンロードフォルダの整理

### __【30\-48分】 複合タスク体験：読み込み → 加工 → 文書生成 を一気に__

- __フェーズ:__ ハンズオン（メインパート）
- __スライド内容:__
	- __タイトル:__ 複合タスク ── 「データ処理＋文書生成」を1回の指示で

- __講師スクリプト案:__  
「これまで、文書生成とデータ処理を別々に体験しました。今日のメインは**両方を組み合わせた複合タスク**です。実際の業務では、データを集計してからレポートを書くことが多いですよね。これを1回の指示でまとめて実行します。」

- __共通お題:__

> **Cowork へのタスク入力（コピーして使用）：**  
> 「デスクトップの『研修出力』フォルダにある `配送実績_元データ.csv` を使って、以下の2つの成果物を同じフォルダに作成してください。  
>  
> ■ 成果物①：`配送実績_月次集計.xlsx`  
> - 元データを配送先別に集計  
> - 合計金額の降順で並べる  
> - 合計行を最終行に追加（SUM関数）  
> - グラフ（配送先別金額の棒グラフ）を集計シートに追加  
>  
> ■ 成果物②：`月次レポート_上司向け.txt`  
> - 上司向けの報告メール文面  
> - 集計結果の要点を3行で箇条書き（例：最も配送が多かった取引先、合計金額等）  
> - 丁寧なビジネス文体  
> - 300文字以内」

- __操作手順:__
	1. 先ほど作成したプロジェクト内でタスクを入力（プロジェクト指示が自動参照される）
	2. Claude が複数の成果物を並行生成する過程を観察（サブエージェントの動き）
	3. 完了後、2つのファイルを開いて確認

- __成果の共有（5分）:__  
「同じ指示を出しても、レポートの文面が少し違う場合があります。隣の人と比べてみてください。良い表現があれば参考にしましょう。修正したい部分があれば追加指示を出してみてください。」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「プロジェクトの指示は後から変えられますか？」__
	- __A:__ 「いつでも編集できます。プロジェクトの設定画面から指示を更新すれば、次回以降のタスクに反映されます。運用しながら改善していくのがベストです。」

- __Q2: 「スケジュールタスクは、PCを閉じても動きますか？」__
	- __A:__ 「いいえ、PCが起動中かつ Claude Desktop が開いている間だけ動きます。PCがスリープになると停止します。毎日の定期タスクを使いたい場合は、出勤時に Claude Desktop を起動しておく運用がおすすめです。」

- __Q3: 「複合タスクが途中で失敗したらどうなりますか？」__
	- __A:__ 「途中で失敗した場合、Claude がエラーの原因を表示します。『○○のファイルが見つかりません』等のメッセージを見て、修正指示を出してください。完了した部分はそのまま残ります。」

- __Q4: 「第06回までに何を準備すればいいですか？」__
	- __A:__ 「棚卸しシートの AI化候補から**1つ**を選んで、今日のフローチャートで仮の方針を出しておいてください。完璧でなくてOKです。第06回の冒頭で講師と相談しながら最終決定します。」
