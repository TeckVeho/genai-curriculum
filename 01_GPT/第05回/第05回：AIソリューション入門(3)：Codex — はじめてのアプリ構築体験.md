# __第05回：AIソリューション入門 (3)：Codex — はじめてのアプリ構築体験__

---

## __1\. 講義概要__

- __フェーズ:__ 座学（実践体験）
- __所要時間:__ 60分
- __主要機能:__ Codex Web（chatgpt.com/codex）
- __形式:__ オンライン

### __狙い__

全員が**共通お題で動くWebアプリを1つ作り上げる**成功体験を持つ。特に「段階的に指示を出すことでアプリが育っていく」プロセスを体感し、第06回以降のワークショップで自分の業務課題に応用できる「Codexへの指示力」を身につける。

### __到達目標__

1. Codexに段階的に指示を出し、入力フォーム → 一覧表示 → UIの改善まで完成させられる
2. 「同じ指示でも出力が異なる」ことを体験し、指示の具体性の重要性を実感している
3. Skills vs Codex の判断フローチャートを理解し、自分の業務課題にどちらが向くかイメージできる
4. 第06回以降のワークショップで取り組む課題を具体的に決められる

### __準備物（講師）__

- プロジェクター・スクリーン（またはオンライン画面共有環境）
- 講師用PC（GitHub・ChatGPT Plus・Codex Web を開いておく）
- 配布資料：**Skills vs Codex 判断フローチャート**（本テキスト4節参照）
- 事前に共通お題のアプリを Codex で構築しておく（デモ完成版として）

### __準備物（受講者）__

- PC（ブラウザ Chrome または Edge 推奨）
- ChatGPT アカウント（Plus 推奨）
- GitHub アカウント・Codex Web 連携済み（第04回で設定完了済み）
- 前回の棚卸しシート・第04回テイクアウェイシート（手元に置いておく）

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

---

### __【00\-10分】 前回の振り返り & Codex プロンプティングのコツ__

- __フェーズ:__ 復習・準備
- __スライド内容:__
	- __タイトル:__ 「良い指示がアプリの質を決める」── Codex プロンプティング 3つのコツ

- __講師スクリプト案:__  
「前回はCodexの環境を整え、小さなタスクを1つ実行しました。今日はいよいよ**アプリを作ります**。  
まず、Codexへの指示で覚えておきたい3つのコツを確認します。」

- __Codexプロンプティング 3つのコツ（スライド）:__

	1. **一度に全部頼まない** ── まず最小構成で動かし、その後に機能を追加していく（例：最初はフォームだけ、次に一覧表示を追加）
	2. **完成イメージを具体的に書く** ── 「入力欄は〇〇・〇〇・〇〇の3つ」「一覧は表形式で」のように詳細に
	3. **うまくいかなければ修正指示を重ねる** ── 一発で完璧を目指さない。「〇〇が表示されていません。追加してください」と会話を続ける

---

### __【10\-45分】 共通お題ハンズオン：業務日報入力フォームを作る__

- __フェーズ:__ ハンズオン（メインパート）
- __スライド内容:__
	- __タイトル:__ 共通お題 ── 業務日報入力フォームを3段階で作ろう

- __講師スクリプト案:__  
「今日の共通お題は**業務日報入力フォーム**です。物流現場でよく使う日報アプリを、3段階に分けて作ります。講師の画面と同じ操作をしながら進めてください。」

---

#### __Step 1：入力フォームだけ作る（10分）__

> **Codex へのタスク入力（コピーして使用）：**  
> 「シンプルなWebアプリを作ってください。日報を入力するフォームです。入力欄は次の4つです：①氏名（テキスト）、②日付（日付入力）、③本日の業務内容（テキストエリア）、④業務に費やした時間（数字、単位は時間）。送信ボタンを押すとアラートで『送信しました』と表示されるだけでOKです。シンプルで見やすいデザインにしてください。」

- 送信 → 実行ログを観察 → Changes タブで確認 → Apply
- ブラウザでプレビューを開いてフォームが表示されることを確認

---

#### __Step 2：一覧表示機能を追加する（10分）__

> **修正指示：**  
> 「送信ボタンを押したとき、アラートではなく、入力した内容をフォームの下に表形式で追記表示するように変更してください。表のヘッダーは『氏名・日付・業務内容・時間』にしてください。ページを更新しても表が消えないよう、ブラウザのローカルストレージに保存してください。」

- 修正後の Changes を確認 → Apply
- フォームに入力して送信 → 一覧に表示されることを確認

---

#### __Step 3：見た目を整える（10分）__

> **修正指示：**  
> 「デザインを改善してください。①フォーム全体に薄いグレーの背景色をつける、②タイトルを大きく『業務日報』と表示する、③送信ボタンを青色にする、④表の行を交互に色をつけて読みやすくする。」

- 最終的な完成物を全員でスクリーンショットまたはプレビュー表示して確認

- __成果の共有（5分）:__  
「同じ指示を出しても、出力が少し違う場合があります。それぞれの画面を見せ合ってください。良いと思ったデザインや機能があれば、修正指示として追加してみましょう。」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「今日作ったアプリは、実際の業務で使えますか？」__
	- __A:__ 「今日のアプリはデータがブラウザのローカルストレージに保存されるため、そのPCでしか見られません。実際の業務で使うには、サーバーへのデプロイや共有設定が必要です。これは第09回の仕上げ回で対応します。」

- __Q2: 「Codex が作ったコードを自分で修正することはできますか？」__
	- __A:__ 「できます。ただし自分でコードを読んで直すのは難しい場合があります。その場合は『〇〇の部分をこう変えてください』とCodexに修正指示を出すのが最も効率的です。コードを知らなくても、日本語の指示で大半の変更に対応できます。」

- __Q3: 「Skills にするか Codex にするか、第06回までに決められそうにありません」__
	- __A:__ 「大丈夫です。第06回の冒頭で、講師と一対一で相談しながら決めます。今日の判断フローチャートで仮の候補を考えておいてくれれば十分です。」

- __Q4: 「Codex で作れないアプリはありますか？」__
	- __A:__ 「高度なシステム連携（既存の基幹システムとの統合）や、リアルタイム処理が必要な複雑なシステムは難しい場合があります。ただし、この研修で目指す『事務業務を楽にするシンプルなWebアプリ』の範囲であれば、ほぼ対応可能です。」
