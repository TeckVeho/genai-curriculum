# __第04回：AIソリューション入門 (2)：Codex — 概念理解・環境構築・基本操作__

---

## __1\. 講義概要__

- __フェーズ:__ 座学
- __所要時間:__ 60分
- __主要機能:__ Codex Web（chatgpt.com/codex）
- __形式:__ オンライン

### __狙い__

IT初心者にとって最大のハードルである「**環境構築**」をこの回でクリアする。Codexとは何かを概念レベルで理解し、GitHub連携 → Codex Web接続 → 小さなタスクの実行まで、**「Codexが動く状態」を全員が確実に整える**。次回のアプリ構築体験を「作ることだけに集中できる状態」で迎えるための準備回。

### __到達目標__

1. Codex が「何をするツールか」を自分の言葉で説明できる（Skills との違いも含め）
2. GitHub アカウントを作成し、Codex Web（chatgpt.com/codex）と連携できる
3. Codex に小さなタスクを依頼し、「指示する → 待つ → 確認する → 採用/やり直し」のサイクルを1回完了できる
4. 次回のアプリ構築体験に向けた見通しを持てる

### __準備物（講師）__

- プロジェクター・スクリーン（またはオンライン画面共有環境）
- 講師用PC（GitHub アカウント・ChatGPT Plus・Codex Web を事前に開いておく）
- 配布資料：**GitHub アカウント作成手順書**（スクリーンショット付き。本テキスト4節参照）
- 配布資料：**Codex Web 環境構築手順書**（スクリーンショット付き。本テキスト4節参照）
- TA（ティーチングアシスタント）が環境構築サポートに入れると理想（ITトラブルが多い回）

### __準備物（受講者）__

- PC（ブラウザ Chrome または Edge 推奨）
- ChatGPT アカウント（Plus 推奨。無料版は Codex 利用枠が制限される場合あり）
- GitHub アカウント（前回の宿題。未作成の場合は今回の前半で作成）

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

---

### __【00\-10分】 Codex とは何か（概念説明）__

- __フェーズ:__ 概念説明
- __スライド内容:__
	- __タイトル:__ Codex ── 自然言語でアプリを作るコーディングエージェント
	- __キーメッセージ:__ 「プログラミングを知らなくてもアプリが作れる」を実現するツール

- __講師スクリプト案:__  
「今日から2回かけて **Codex** を学びます。  

Codexを一言で言うと、**『ChatGPTに"こんなアプリを作って"と日本語で頼むと、クラウド上で本当に動くアプリを作ってくれるツール』**です。  

前回のSkillsとの違いを整理します。  
- **Skills：** ChatGPTとの会話の中で完結する。定型業務を自動化する。  
- **Codex：** 独立したアプリ（Webアプリ）を作る。フォーム・一覧表示・データ管理等が必要な業務に向いている。  

物流で例えると、Skillsは『倉庫スタッフに業務マニュアルを渡す』こと。Codexは『そのスタッフが使う業務管理システムを新しく作る』こと。どちらを使うかは、業務課題の性質で第06回に判断します。」

- __Codexの仕組み（シンプルに）:__
	1. chatgpt.com/codex にアクセス
	2. 日本語でタスクを書いて送信
	3. Codexがクラウドサンドボックス（インターネット上の作業環境）でコードを生成・実行
	4. 結果をレビュー画面で確認 → 採用 or やり直し
	5. GitHubに保存・共有

---

### __【10\-35分】 環境構築ハンズオン__

- __フェーズ:__ ハンズオン（最重要パート）
- __スライド内容:__
	- __タイトル:__ まず「動く環境」を整えましょう ── GitHub 連携手順

- __講師スクリプト案:__  
「Codexを使うには、**GitHub（ギットハブ）** というサービスのアカウントが必要です。GitHubは、作ったアプリのコードを保存・管理する場所です。自分では一行もコードを書きませんが、Codexが作ったコードの置き場所として使います。  
では手順書に沿って進めましょう。詰まったら手を挙げてください。」

- __Step 1：GitHub アカウントの確認・作成（5分）__
	1. [https://github.com](https://github.com) にアクセス
	2. 「Sign up」→ メールアドレス・パスワード・ユーザー名を入力
	3. メール認証を完了させる
	4. 講師が画面を映しながら進め、受講者が並走する形で実施

- __Step 2：Codex Web にアクセス（5分）__
	1. [https://chatgpt.com/codex](https://chatgpt.com/codex) にアクセス
	2. ChatGPT アカウントでログインされていることを確認
	3. 画面の構成を確認：タスク一覧パネル・タスク作成ボタン・実行結果エリア

- __Step 3：GitHub 連携（10分）__
	1. Codex Web 画面の「GitHub を接続する」ボタンをクリック
	2. GitHub の認証画面が開く → 「Authorize」をクリック
	3. 接続完了の確認
	4. 新しいリポジトリの作成：「+ New repository」→ 名前を `my-work-tools` に設定 → Public → Create
	5. Codex Web でそのリポジトリが選択できることを確認

- __トラブルシューティング（よくある問題）:__
	- GitHub のメール認証が来ない → 迷惑メールフォルダを確認
	- Codex Web に「Plus プランが必要」と表示される → ChatGPT のアカウント設定を確認。無料版の場合は講師アカウントとペアで進める
	- 「Authorize」後に画面が戻らない → ブラウザを更新

### __【35\-53分】 基本操作体験：Codexのサイクルを1回回す__

- __フェーズ:__ 操作体験
- __スライド内容:__
	- __タイトル:__ Codexの操作フロー ── 「指示 → 待つ → 確認 → 採用/やり直し」

- __講師スクリプト案:__  
「環境ができたら、Codexに小さなタスクを出してみましょう。今日のお題は超シンプルです。アプリではなく、まず『Codexがどう動くか』を体感することが目的です。」

- __共通お題（全員同じタスクを実行）:__

> **タスク入力例：**  
> 「README.md というファイルを作って、次の内容を日本語で書いてください：①このリポジトリの名前（my-work-tools）、②このリポジトリの目的（業務効率化ツールを作成・管理する場所）、③作成日（今日の日付）」

- __操作手順:__
	1. Codex Web の「New Task」ボタンをクリック
	2. 上記テキストを入力欄に貼り付けて送信
	3. Codex がクラウドで処理中（数十秒〜数分）→ 実行ログが流れる様子を観察
	4. 完了したら「Changes」タブで変更内容（README.md の中身）を確認
	5. 問題なければ「Apply」ボタンで変更を採用
	6. GitHub リポジトリにファイルが追加されていることをブラウザで確認

- __解説ポイント:__
	- 「実行ログが流れている間、Codexがクラウドでコードを書いています。私たちは待つだけです」
	- 「Changesタブが『変更内容のレビュー画面』。Apply前に必ず中身を確認する習慣を」
	- 「うまくいかなかった場合は『〇〇が足りません。追加してください』と修正指示を出してください」

- __修正指示の練習（時間があれば）:__

> **修正指示例：**  
> 「README.md に、このリポジトリの使い方（初心者向けに3ステップで）も追加してください」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「GitHub って何ですか？プログラマーが使うものではないですか？」__
	- __A:__ 「本来はプログラマーがコードを管理するサービスです。しかし今回の研修では、Codexが作ったアプリの『保存先』として使います。皆さんが自分でコードを書くことはありません。フォルダを作ってファイルを保存するイメージで大丈夫です。」

- __Q2: 「Codex が作ったアプリは、社内のパソコンで使えますか？」__
	- __A:__ 「Codexが作ったWebアプリはブラウザで動きます。インターネットに公開する設定（デプロイ）をすれば、URLを知っている人なら社内外問わず使えます。セキュリティが必要な場合はアクセス制限を設けることもできます。第09回の仕上げ回で対応します。」

- __Q3: 「無料版の ChatGPT でも Codex は使えますか？」__
	- __A:__ 「無料版でも一部使えますが、利用回数に制限があります。研修期間中は Plus プランを推奨します。研修終了後に継続するかどうかは、使い続けるかどうかで判断してください。月額約3,000円です。」

- __Q4: 「Codex が作ったコードは安全ですか？」__
	- __A:__ 「Codexはクラウドのサンドボックス（隔離された環境）で動作するので、皆さんのPCには直接影響しません。ただし、作ったアプリをインターネットに公開する場合は、個人情報や機密データを扱わない設計にすることが大原則です。第06回の設計ワークでこの点も確認します。」
