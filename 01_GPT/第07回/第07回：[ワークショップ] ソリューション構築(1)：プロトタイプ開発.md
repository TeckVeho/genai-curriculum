# __第07回：[ワークショップ] ソリューション構築 (1)：プロトタイプ開発__

---

## __1\. 講義概要__

- __フェーズ:__ 実践
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第06回で作成した設計書をもとに、**実際に動くプロトタイプを作る**。Codex 組は「入力フォーム + 一覧表示」という最小構成のWebアプリを、Skills 組は「テスト実行まで完了した基本Skill」を完成させることがゴール。「完璧にしようとせず、まず動かす」という姿勢を定着させる。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. 設計書をAIへの指示文として変換し、Codex / Skills に投入できる
2. プロトタイプ（最小動作版）を1つ完成させられる
3. 「思い通りに動かないとき」に修正指示を出せる
4. 次回（第08回）で改善すべき点を3つ以上書き出せる

### __準備物（講師）__

- PC（GitHub・ChatGPT Plus・Codex Web を開いておく）
- 第06回で各受講者が作成した設計書（講師もコピーを持っておくと巡回指導がしやすい）
- トラブルシューティング参考資料（Codex のよくあるエラーと対処法）

### __準備物（受講者）__

- PC（GitHub・ChatGPT アカウント利用可能な状態）
- 第06回で作成したアプリ仕様書（Codex 組）または Skill 定義書（Skills 組）

### __【05\-50分】 構築ワーク（Codex 組 / Skills 組 並行）__

- __フェーズ:__ 個人作業（講師は両組を巡回）

---

#### __Codex 組：プロトタイプ Webアプリの構築__

**Step 1：設計書をCodexへの指示文に変換する（10分）**

設計書の内容を、Codexに渡せる形の日本語指示文に書き直す。以下のテンプレートを参考に入力欄に記入する：

```
シンプルなWebアプリを作ってください。
アプリ名：[アプリ名]

■ 入力フォームの項目：
- [項目1]（[入力タイプ：テキスト/数字/日付等]）
- [項目2]
- [項目3]
※ 送信ボタンを押すと、入力した内容をフォーム下の表に追記表示する

■ 表（一覧）の仕様：
- ヘッダー：[項目1]・[項目2]・[項目3]
- データはブラウザのローカルストレージに保存（ページ更新後も消えない）

■ デザイン：シンプルで見やすいものにしてください。
```

**Step 2：Codex に投入する（15分）**

1. Codex Web（chatgpt.com/codex）を開き「New Task」
2. 作成した指示文を貼り付けて送信
3. 実行ログを観察しながら待つ
4. Changes タブで生成されたコードの概要を確認
5. Apply して動作確認

**Step 3：動作確認と修正指示（10分）**

プレビューまたはブラウザでアプリを開き、以下を確認する：

- [ ] フォームの全項目が表示されている
- [ ] 送信ボタンを押すと一覧に追記される
- [ ] ページを更新してもデータが残る

問題があれば修正指示を出す（例：「送信ボタンが見当たりません。追加してください」）

---

#### __Skills 組：基本 Skill の構築__

**Step 1：Skill 定義書をもとに指示文を完成させる（10分）**

第06回の定義書の「指示文の下書き」を、以下の観点で確認・補強する：

- 役割（AIがどんな立場で動くか）は明記されているか
- 出力形式（文体・文字数・含めるべき要素）は具体的か
- 入力がどの形式で来るかを指示文に含めているか

**Step 2：Skill を作成する（15分）**

1. ChatGPT の Skills 画面を開き「新しい Skill を作成」
2. Skill 名・説明・Instructions（指示文）を入力
3. 保存する

**Step 3：テスト実行と修正（10分）**

定義書のテスト用入力例を使って Skill を実行する。

- [ ] 期待通りの出力が返ってきているか
- [ ] 文体・文字数・含めるべき要素は揃っているか
- [ ] 想定外の出力がある場合 → Instructions の該当箇所を修正

修正例：「出力が長すぎる → Instructions に『300文字以内』を追加」

---

- __講師の巡回ポイント:__
	- 「Codexが動かない」→ 指示文が曖昧でないか確認。具体的な項目名・動作を追加するよう促す
	- 「Skillの出力がおかしい」→ Instructions の条件を1つずつ確認し、足りていない制約を追加
	- 「何から手をつければいいかわからない」→ 「まず設計書の1番目の項目だけでいいです。入力フォームだけ / Skill名だけから始めましょう」
	- 環境接続に問題がある受講者のフォロー（Codex連携エラー等）

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「Codex がエラーを返してきました。どうすればいいですか？」__
	- __A:__ 「エラーメッセージをそのままCodexに渡して『このエラーを解決してください』と指示してください。Codexは自分が出したエラーを自分で直すのが得意です。」

- __Q2: 「プロトタイプが完成しませんでした」__
	- __A:__ 「大丈夫です。今日完成しなかった分は次回の冒頭15分で追いつく時間を設けます。今日の作業内容と、どこで詰まったかをメモしておいてください。」

- __Q3: 「Skills の出力が毎回少し違います。安定させるにはどうすればいいですか？」__
	- __A:__ 「Instructions に出力のパターンを具体的に書くと安定します。例：『必ず以下の構成で出力してください：①件名 ②本文（3段落） ③締めの一文』のように構造を明示してください。」
