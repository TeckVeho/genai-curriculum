# __第09回：[ワークショップ] ソリューション構築 (3)：仕上げ・テスト・共有__

---

## __1\. 講義概要__

- __フェーズ:__ 実践（最終仕上げ）
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第07〜08回で構築したソリューションを**発表できる状態に仕上げる**。Codex 組はアプリの最終調整・テスト・共有設定を、Skills 組は最終テスト・チーム共有設定・運用マニュアル作成を完了させる。また発表会（第10回）に向けた**デモ手順・説明資料**を準備する。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. ソリューションを「誰かに渡せる状態（共有可能・テスト済み）」に完成させられる
2. 3行運用マニュアル（使い方のメモ）を作れる
3. 第10回の発表に向けて Before/After・デモ手順を準備できる
4. 自分が作ったソリューションの限界・今後の改善点を言語化できる

### __準備物（講師）__

- PC（GitHub・ChatGPT Plus・Codex Web を開いておく）
- 第10回の発表フォーマット（本テキスト4節参照）
- Codex 組向け：GitHub Pages または簡易デプロイ手順書（任意。共有 URL を発行できるようにする）

### __準備物（受講者）__

- PC（GitHub・ChatGPT アカウント利用可能な状態）
- 第07〜08回で作ったソリューション（保存済みの状態）
- 第08回テイクアウェイの「次回タスクリスト」「発表会準備メモ」

---

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

---

### __【00\-05分】 本日のゴールと優先順位の確認__

- __スライド内容:__
	- __タイトル:__ 今日のゴール：「誰かに渡せる状態」にする

- __講師スクリプト案:__  
「今日は最終仕上げです。ゴールは**第10回の発表会でデモできる状態に完成させること**です。  
優先順位はこの順番です：①動く・使える状態にする、②共有設定（URLや共有リンクを発行する）、③発表資料を作る。見た目の細かい調整は最後です。」

---

### __【05\-40分】 仕上げ作業（Codex 組 / Skills 組 並行）__

- __フェーズ:__ 個人作業（講師は両組を巡回）

---

#### __Codex 組：アプリの最終仕上げと共有__

**作業リスト（優先順に取り組む）:**

**① 残機能の完成（第08回タスクリストの残り）**  
未完了の改善項目を Codex への指示で対応する。

**② 動作テスト（必須）**  
以下のテストシナリオで動作確認を行う。
- [ ] フォームの全項目に正しく入力できるか
- [ ] 送信ボタンで一覧にデータが追加されるか
- [ ] ページ更新後もデータが保持されるか
- [ ] 想定外の入力（空欄・数字欄に文字など）でエラーが起きないか

**③ 3行運用マニュアルの作成**  
Codexに以下を指示して作成：

> 「このアプリの使い方を、PC操作が苦手な社員向けに3ステップで説明する文章を書いてください。各ステップは1文で、小学生にもわかる言葉で書いてください。」

**④ 共有設定（任意：時間があれば）**  
GitHub Pages を使ってアプリを URL で共有できる状態にする（手順書に従って設定）。

---

#### __Skills 組：Skill の最終仕上げとチーム共有__

**作業リスト（優先順に取り組む）:**

**① 最終テスト（必須）**  
以下のテストシナリオで動作確認を行う。
- [ ] 通常の入力パターンで期待通りの出力が返るか
- [ ] 入力が少ない・不完全なケースで適切に対応するか
- [ ] 第08回で追加した条件が正しく機能しているか

**② 3行運用マニュアルの作成**  
ChatGPTに以下を指示して作成：

> 「この Skill の使い方を、IT が苦手な同僚向けに3ステップで説明する文章を書いてください。各ステップは1文で、具体的な操作を含めてください。」

**③ チーム共有設定（Team / Business プランの場合）**  
Skills の「共有」設定から、チームメンバーへの配布設定を行う。

---

### __【40\-50分】 発表準備（全員共通）__

- __フェーズ:__ 発表準備
- __スライド内容:__
	- __タイトル:__ 第10回の発表準備をしよう

- __講師スクリプト案:__  
「残り20分で発表準備をします。第10回は1人あたり**2〜3分**の発表です。以下の順番で話してください。」

- __発表フォーマット（スライドで提示・配布）:__

	1. **業務課題（Before）：**  
	「私は毎週〇〇という業務に△△時間かかっていました」
	
	2. **作ったソリューション：**  
	「〇〇（Skills / Codex で作った〇〇アプリ / Skill）を使って解決しました」
	
	3. **デモ（30〜60秒）：**  
	実際に画面を見せながら動かす（入力 → 出力 or アプリ操作を1回）
	
	4. **After（見込み）：**  
	「これを使うと、週〇〇時間 → 〇〇時間に短縮できる見込みです」
	
	5. **今後の展望・感想：**  
	「次はこんな改善をしたい」「研修で特に役立ったこと」を一言

- __発表準備ワーク（10分）:__
	- 発表フォーマットに沿ってメモを作成
	- デモ手順を1回通しで確認する（失敗しても大丈夫、練習なので）

---

### __【50\-55分】 グループ内リハーサル（任意）__

- __スライド内容:__
	- __タイトル:__ 隣の人に1分で発表してみよう

- 2人1組で発表フォーマットを使って1分間のリハーサル
- 「デモ画面が見えるか」「話の流れがわかるか」を相互フィードバック

	- __タイトル:__ 今日のフィードバックにご回答ください（3分）
	- QRコード または URL を大きく表示

- __アンケート項目:__
	- 詳細は **[アンケート/各回フィードバックアンケート.md](../アンケート/各回フィードバックアンケート.md)** を参照
	- 理解度（5段階）／業務活用イメージ（5段階）／進行スピード（5段階）／満足度（5段階）／自由記述

次回はいよいよ**発表会**です。今日作った発表メモとデモ手順を確認しておいてください。当日は順番に画面を共有しながらデモをしてもらいます。  
最後に**事後アンケート**もあります。事前アンケートの内容を思い出しながら、どれだけ変わったかを振り返っておいてください。」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「アプリがまだ完成していません。第10回までに間に合いますか？」__
	- __A:__ 「第10回の冒頭に10分の最終作業時間を設けます。ただし完成度より『何を作ったか・何を解決しようとしたか』の方が大切です。動かない部分があっても、課題・設計の意図・今後の方向性を発表できれば十分です。」

- __Q2: 「発表が苦手で緊張します」__
	- __A:__ 「発表会は競争ではありません。同じ受講者同士、お互いの業務を知り合う場です。『ここがうまくできた』も『ここで詰まった』も、両方話してOKです。」

- __Q3: 「作ったアプリを研修後も使い続けることはできますか？」__
	- __A:__ 「Codex 組のアプリは GitHub に保存されているので、自分のアカウントで引き続き使えます。改善したい場合は Codex に追加指示を出すことで、研修後も育てられます。Skills 組も同様に、ChatGPT アカウントが有効な限り使えます。」
