GitHub Pages 簡易デプロイ手順書

利用タイミング： 第09回の仕上げ作業（任意）で使用します。
Codex 組が作ったWebアプリを、URLで他の人と共有できる状態にします。
スキップしても発表会への参加・デモは可能です。

GitHub Pages とは

  • GitHub のリポジトリにある HTML/CSS/JavaScript ファイルを、無料でWeb公開できる機能
  • 公開後は https://ユーザー名.github.io/リポジトリ名/ というURLでアクセス可能
  • 設定後すぐ（〜数分）で公開される

手順

Step 1：GitHub でリポジトリを開く

  1. ブラウザで https://github.com にアクセス
  2. 右上のアイコン → 「Your repositories」をクリック
  3. my-work-tools（または作成したリポジトリ名）をクリック

Step 2：Settings（設定）を開く

  1. リポジトリ画面の上部タブから 「Settings」 をクリック
  2. 左側メニューを下にスクロールして 「Pages」 をクリック

Step 3：GitHub Pages を有効にする

  1. 「Source」 のドロップダウンで 「Deploy from a branch」 を選択
  2. Branch のドロップダウンで 「main」 を選択
  3. フォルダのドロップダウンで 「/ (root)」 を選択
  4. 「Save」 をクリック

Step 4：公開URLを確認する

  1. 設定後1〜3分待つ

  2. 同じ「Pages」設定画面に戻ると、以下のような表示が出る：

    Your site is live at:
    https://あなたのユーザー名.github.io/my-work-tools/

  3. リンクをクリックしてアプリが表示されるか確認する

✅ チェック： URLでアプリが開いた → □ 完了

Step 5：URLを発表・共有に使う

  • 発表会（第12回）でこのURLをスクリーンに映してデモすることができます
  • チームメンバーへの共有もこのURLで可能（ログイン不要でアクセス可能）

よくある問題

症状 対処法
URLにアクセスすると「404」が表示される index.html がリポジトリのルートにあるか確認。Codex で「index.html をルートディレクトリに配置してください」と指示
設定画面に Pages が表示されない リポジトリが Public（公開）になっているか確認。Private の場合は Settings → General → Change repository visibility から変更
ページが表示されるがCSSが当たっていない（デザインが崩れる） CSSファイルのパスが相対パスになっているか Codex に確認させる
更新しても古いバージョンが表示される ブラウザのキャッシュをクリア（Ctrl+Shift+R または Cmd+Shift+R）

注意事項

⚠️ 個人情報・機密情報をアプリに入力しないでください
GitHub Pages で公開したアプリのURLを知っている人は誰でもアクセスできます。
今回のアプリは「業務フロー・入力項目の構成」のデモ用途で使用してください。

⚠️ ローカルストレージのデータは各ブラウザに保存されます
GitHub Pages で公開しても、入力データは各自のブラウザに保存されます。
複数人でリアルタイムにデータを共有するには、追加のバックエンド設定が必要です（今回のスコープ外）。

デプロイ後の確認チェックリスト

  • GitHub Pages の設定が完了した
  • 公開URL（https://ユーザー名.github.io/...）でアプリが表示された
  • フォーム入力 → 一覧表示の基本動作が確認できた
  • URLを発表準備シートにメモした

★ お疲れ様です！ 公開URLを発表準備シートの「デモのURL」欄に記入しておきましょう。
