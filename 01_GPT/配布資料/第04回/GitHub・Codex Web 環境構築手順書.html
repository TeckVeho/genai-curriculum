GitHub・Codex Web 環境構築手順書

利用タイミング： 第03回末〜第04回で使用します。
この手順書に沿って操作すれば、プログラミング未経験でも環境構築が完了します。
詰まった箇所には✏️でメモを取りながら進めましょう。

Step 1：GitHub アカウントの作成

すでに作成済みの方は Step 2 へ進んでください。

1-1. GitHub にアクセスする

  1. ブラウザ（Chrome または Edge 推奨）を開く
  2. アドレスバーに https://github.com と入力してアクセス
  3. 右上の 「Sign up」 ボタンをクリック

1-2. アカウント情報を入力する

以下の情報を順番に入力します：

入力項目 入力内容 メモ欄
メールアドレス 普段使っているメールアドレス
パスワード 8文字以上（英数字混在推奨）
ユーザー名 英数字・ハイフンのみ（例：tanaka-kohei）
メール受信設定 どちらでも可（スキップ可）

⚠️ ユーザー名は後から変更が難しいため、わかりやすい名前にしましょう。

1-3. ロボット認証を完了する

  • 「パズルを解いてください」という画面が出る場合、指示に従って操作する

1-4. メール認証を完了する

  1. 登録したメールアドレスに 「GitHub メール認証コード」 が届く
  2. コード（8桁の数字）を GitHub の画面に入力する
  3. 迷惑メールフォルダに届いていない場合は「再送信」ボタンを使う
  • メール認証が完了した

1-5. プラン選択

  • 「Free」プランを選択（無料）してください
  • 課金は不要です

1-6. ログイン確認

  • https://github.com にアクセスして、自分のアイコンが右上に表示されていれば OK

✅ チェック： 自分のアイコン（またはアバター）が右上に表示されている → □ 完了

Step 2：Codex Web へのアクセスと GitHub 連携

2-1. Codex Web にアクセスする

  1. ブラウザで https://chatgpt.com/codex にアクセス
  2. ChatGPT アカウントでログインされていることを確認
  3. ログインしていない場合は https://chatgpt.com でログインしてから再度アクセス

✅ チェック： Codex Web の画面（タスク一覧）が表示された → □ 完了

2-2. 画面の構成を確認する

Codex Web の主な画面要素：

画面要素 場所 役割
タスク一覧パネル 左側 過去に実行したタスクの一覧
「New Task」ボタン 上部または中央 新しいタスクを作成・入力する
実行結果エリア 右側・中央 タスクの実行ログ・変更内容を確認する
リポジトリ選択 上部 作業対象の GitHub リポジトリを選択する

2-3. GitHub と Codex Web を連携する

  1. Codex Web 画面内の 「GitHub を接続する」 または 「Connect GitHub」 ボタンをクリック
  2. GitHub の認証画面が開く
  3. 「Authorize」（認可） ボタンをクリック
  4. Codex Web の画面に戻り、連携完了のメッセージを確認する

✅ チェック： GitHub との連携が完了した → □ 完了

2-4. リポジトリを作成する

  1. GitHub（https://github.com）にアクセス
  2. 右上の 「+」アイコン → 「New repository」 をクリック
  3. 以下の通り設定する：
設定項目 入力値
Repository name my-work-tools
Description （任意）業務効率化ツールを作成・管理する場所
Visibility Public（公開）
Initialize this repository 「Add a README file」にチェック
  1. 「Create repository」 をクリック

✅ チェック： my-work-tools リポジトリが作成された → □ 完了

2-5. Codex Web でリポジトリを選択する

  1. Codex Web に戻り、リポジトリ選択画面で my-work-tools を選択
  2. 選択できない場合は画面をリロードする

✅ チェック： Codex Web で my-work-tools が選択できた → □ 完了

Step 3：動作確認（小さなタスクを実行）

3-1. 最初のタスクを実行する

  1. Codex Web で 「New Task」 をクリック
  2. 以下のテキストをコピーして貼り付け、送信する：
README.md というファイルを作って、次の内容を日本語で書いてください：
①このリポジトリの名前（my-work-tools）
②このリポジトリの目的（業務効率化ツールを作成・管理する場所）
③作成日（今日の日付）
  1. 実行ログが流れる様子を観察する（数十秒〜数分）

3-2. 結果を確認する

  1. 処理が完了したら 「Changes」タブ をクリック
  2. README.md の内容が表示されることを確認
  3. 問題なければ 「Apply」ボタン をクリック
  4. GitHub（https://github.com/自分のユーザー名/my-work-tools）にアクセスして、README.md が追加されていることを確認

✅ チェック： GitHub リポジトリに README.md が追加された → □ 完了

トラブルシューティング

症状 対処法
GitHub のメール認証コードが届かない 迷惑メールフォルダを確認 → 「再送信」ボタンをクリック
Codex Web に「Plus プランが必要」と表示される ChatGPT のアカウント設定を確認。無料版の場合は講師に申し出てペアで進める
「Authorize」後に画面が戻らない ブラウザを更新（F5 または Cmd+R）
リポジトリが Codex Web に表示されない ページをリロード。それでも出ない場合は連携設定を再確認
タスク実行が止まらない 数分待つ。それでも終わらない場合は講師に相談

完了チェックリスト（まとめ）

  • GitHub アカウントを作成した（メール認証まで完了）
  • https://chatgpt.com/codex にアクセスできた
  • Codex Web と GitHub を連携できた
  • my-work-tools リポジトリを作成した
  • Codex にタスクを出して、README.md が作成されることを確認した

全部チェックできたら環境構築完了です！
