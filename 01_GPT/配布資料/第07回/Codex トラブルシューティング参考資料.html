よくあるエラーと対処法

エラー①：タスクが終わらない / ずっとローディング中

症状： タスクを送信してから5分以上経っても完了しない。実行ログが止まっている。

原因：

  • 指示が曖昧すぎてAIが判断できない状態
  • 依存ライブラリのインストールに時間がかかっている
  • ネットワークの一時的な問題

対処法：

  1. 5〜10分待つ（Codex はクラウドで動いているため、PCの負荷には関係なし）
  2. それでも終わらない場合は「Stop」ボタンがあれば押す
  3. 新しいタスクを作成し直す（より具体的な指示で）
  4. ブラウザを更新して Codex Web を再読み込みする

エラー②：「Apply」後にアプリが開かない / 真っ白な画面

症状： Changes タブで確認して Apply したが、ブラウザでプレビューすると真っ白。

原因：

  • JavaScript のエラーがある
  • ファイルが正しく生成されていない
  • index.html が生成されていない

対処法：

  1. 新しいタスクで以下を指示：
    アプリを開くと真っ白な画面になります。コンソールのエラーを確認して修正してください。
  2. Changes タブで index.html ファイルが存在するか確認する
  3. 存在しない場合：
    index.html が見当たりません。アプリのエントリーポイントとなる index.html を作成してください。

エラー③：フォームを送信してもデータが表示されない

症状： フォームに入力して送信ボタンを押しても、一覧テーブルに何も表示されない。

原因：

  • JavaScript のイベントリスナーが正しく設定されていない
  • ローカルストレージへの保存処理が未実装

対処法：

送信ボタンを押しても一覧テーブルにデータが追加されません。
以下を確認して修正してください：
- 送信ボタンにクリックイベントが設定されているか
- フォームデータを取得して配列に追加する処理があるか
- テーブルを再描画する処理があるか
- ローカルストレージに保存・読み込みする処理があるか

エラー④：ページ更新するとデータが消える

症状： フォームで入力・送信して一覧に表示されたが、ページを更新すると消えてしまう。

原因：

  • ローカルストレージへの保存が実装されていない

対処法：

ページを更新するとデータが消えてしまいます。
ブラウザの localStorage を使ってデータを保存し、
ページ読み込み時に localStorage からデータを復元するように実装してください。

エラー⑤：「GitHub 連携エラー」/ リポジトリが選択できない

症状： Codex Web でリポジトリ一覧が表示されない、または空白。

原因：

  • GitHub との連携が切れている
  • リポジトリの作成直後で反映されていない

対処法：

  1. ページをリロードする（F5 または Cmd+R）
  2. それでも表示されない場合：
    • Codex Web の設定 → GitHub 連携を一度解除
    • 再度「GitHub を接続する」から連携し直す
  3. リポジトリ名のスペルミスを確認する（my-work-tools が正しい）

エラー⑥：意図と違う変更をされた

症状： 「ボタンの色を変えて」と指示したら、他の部分も変わってしまった。

原因：

  • 指示が広すぎてAIが余計な変更を加えた

対処法：

【変更してほしいこと】送信ボタンの背景色を青（#0070f3）にしてください。
【変更しないでほしいこと】それ以外のデザイン・機能は一切変更しないでください。

コツ： 「変更してほしいこと」と「変更しないでほしいこと」を明示する

エラー⑦：Codex が同じエラーを繰り返す

症状： 修正指示を出してもまた同じエラーが出る。

原因：

  • 根本原因が特定できていない
  • 指示が抽象的すぎる

対処法：

  1. エラーメッセージをそのままコピーして渡す：
    以下のエラーが出ています。原因を特定して修正してください：
[エラーメッセージをここに貼り付け]
  2. ブラウザの開発者ツール（F12）のコンソールタブを開いてエラー内容を確認する
  3. 「現状の問題点を教えてください」と質問してから修正指示を出す

指示文のコツ（効果的な修正指示の書き方）

悪い例 良い例
「動きません」 「送信ボタンを押しても一覧テーブルに行が追加されません」
「エラーが出ます」 「ブラウザコンソールに TypeError: Cannot read property 'push' of undefined と出ます」
「直してください」 「〇〇の部分だけを修正してください。△△は変えないでください」
「もっとよくして」 「テーブルのヘッダー行の背景色を紺色（#003366）、文字色を白（#ffffff）にしてください」

受講者によく出る質問と回答

Q. Codexが作ったコードは安全ですか？
A. Codex はクラウドのサンドボックスで動作するため、PCには直接影響しません。ただし、作ったアプリをインターネットに公開する場合は、個人情報や機密データを扱わない設計にすることが大原則です。

Q. 作ったコードを自分で修正できますか？
A. できます。GitHub でファイルを直接編集することも可能ですが、「〇〇を〇〇に変えてください」と Codex に修正指示を出す方が効率的です。

Q. Codexのタスク実行回数に制限はありますか？
A. ChatGPT Plus プランの場合、一定の利用制限があります。制限に達した場合は数時間後にリセットされます。頻繁に制限に達する場合はタスクをまとめて効率化することを検討してください。
