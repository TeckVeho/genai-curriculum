## 研修後も使える：Skills vs Codex 判断フローチャート（再掲）

```
Q1. 解決したい業務は「ChatGPTとの会話の中で完結する」か？
  ├─ はい → Q2へ
  └─ いいえ（独立した画面・フォームが欲しい）→ 【Codex】

Q2. 毎回ほぼ同じ手順・判断を繰り返すか？
  ├─ はい → 【Skills】
  └─ いいえ（都度異なる入力・判断が必要）→ Q3へ

Q3. データの蓄積・一覧表示・集計が必要か？
  ├─ はい → 【Codex】
  └─ いいえ → 【Skills】
```

---

## 研修後の次のステップ

1. **今日作ったソリューションを実務で使ってみる**（まず1週間）
2. **不便な点を修正指示で改善する**（「〇〇が使いにくい。こう直してください」と Codex / ChatGPT に伝える）
3. **第2・第3のソリューションを作る**（棚卸しシートの2番・3番の課題へ）
4. **チームに共有する**（Skills の場合は共有設定、Codex の場合は GitHub Pages URL を共有）
