# __第10回：[ワークショップ] 業務適用＋運用設計__

---

## __1. 講義概要__

- __フェーズ:__ 実践
- __所要時間:__ 60分
- __形式:__ オンライン（またはオフライン）

### __狙い__

第09回までに「動く・共有できる」状態にした Skills / Codex ソリューションを、**実務に載せる前提**で整理する。入力データのルール、担当・頻度、例外時の手当て、見直しサイクルをメモ化し、第11回の「仕上げ・社内展開」と第12回の発表で説明しやすい形にする。

### __到達目標__

1. 「誰が・いつ・どのデータで」使うかを1枚に書ける
2. 例外パターン（データ欠損・締切変更など）と、そのときの人の対応を2つ以上想定できる
3. 発表準備シートの Before/After 数字を、業務に即した言い方に言い換えられる
4. 次回（第11回）に向けて残タスク（資料化・社内向け一言説明）を列挙できる

### __準備物（講師）__

- PC（ChatGPT・Codex / GitHub が開ける状態）
- 第09回までの進捗メモ（任意）

### __準備物（受講者）__

- PC（ChatGPT アカウント・必要なら GitHub にログイン済み）
- 第09回までのソリューションと **発表準備シート（ドラフト）**

---

## __2. 講義スクリプト & スライド構成（60分）__

### __【00〜08分】オープニング・本日のゴール__

- __講師スクリプト案:__  
「今日は**業務適用と運用設計**です。完成度を上げるというより、**現場で回すときに必要な決めごと**を埋めます。デモが多少未完成でも、『誰が・いつ・何を入力するか』がはっきりしていると、発表と研修後の運用がぐっと楽になります。」

### __【08〜45分】ワーク（個人作業＋講師巡回）__

受講者は以下を **メモまたはスプレッドシート** に記入する。

| 項目 | 記入内容 |
|------|----------|
| 利用シーン | 週次／月次／不定期、担当役割 |
| 入力データ | 必須項目・取得元・個人情報の有無 |
| 出力先 | メール／ファイル／アプリ内など |
| 正常系 | 期待されるアウトプット |
| 例外系 | よくあるズレと、そのときの人手順 |
| リスク | 誤出力時の確認ポイント |

**Codex 組：** URL 共有や権限の前提を一行メモする。  
**Skills 組：** Skill の呼び出し手順（誰がどの画面から）を一行で書く。

### __【45〜52分】ペアレビュー（任意・短時間）__

隣同士で「担当・頻度・例外」だけ読み合い、わからない点を1つフィードバック。

### __【52〜58分】フィードバックアンケート__

**[アンケート/各回フィードバックアンケート.md](../アンケート/各回フィードバックアンケート.md)**（第10回として回答）

### __【58〜60分】クロージング & 次回予告__

「次回 **第11回** は **成果物の完成と社内展開の準備** です。今日の運用メモをもとに、説明用の一言・簡易マニュアル・発表シートを仕上げていきます。**事後アンケートは第12回**です。」

## __3. Takeaway・次回までの宿題__

- 発表準備シートの **Before/After** を、今日の運用メモと矛盾がないよう見直す
- 社内の人に説明するときの **30秒ピッチ** を3文で書いておく
