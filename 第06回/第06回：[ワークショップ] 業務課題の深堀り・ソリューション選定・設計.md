# __第06回：[ワークショップ] 業務課題の深堀り・ソリューション選定・設計__

---

## __1\. 講義概要__

- __フェーズ:__ 実践（ワークショップ開始）
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第01回の業務棚卸しに立ち返り、座学（第02〜05回）で得た知識を使って**「自分の課題に最適なAIソリューション（Skills or Codex）を自分で選定・設計する」**。この回で決めた設計が第07〜09回の構築の土台になるため、「作るものが明確になった状態」で終わることが目標。各自の設計内容に講師が個別フィードバックする時間も設ける。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. 第01回の棚卸しシートを再評価し、取り組む業務課題を1つに絞れる
2. 判断フローチャートを使い、Skills / Codex どちらを使うかを確定できる
3. 選んだツールに応じて、自然言語でアプリ仕様（Codex組）またはSkill定義（Skills組）を書き出せる
4. 第07回以降で構築を開始できるレベルまで設計を具体化できる

### __準備物（講師）__

- ホワイトボード・マーカー（ソリューション選定の板書用）
- 配布資料：**Codex 組：アプリ仕様書テンプレート**（本テキスト4節参照）
- 配布資料：**Skills 組：Skill 定義書テンプレート**（本テキスト4節参照）
- 第01回の業務棚卸しシート（受講者が持参するが、予備を印刷しておく）
- Skills vs Codex 判断フローチャート（再配布用）
- 付箋・ペン（課題の書き出しワーク用）

### __準備物（受講者）__

- PC（GitHub・ChatGPT アカウント利用可能な状態）
- 第01回の業務棚卸しシート
- 第05回の Skills vs Codex 判断フローチャート（仮の答えを記入済み）

---

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

---

### __【00\-10分】 棚卸しシートの再確認と課題の絞り込み__

- __フェーズ:__ 振り返り・絞り込み
- __スライド内容:__
	- __タイトル:__ 第01回の棚卸しシートを再評価しよう

- __講師スクリプト案:__  
「ここからの4回は**自分の業務課題を解決するものを実際に作る**時間です。まず第01回の棚卸しシートを開いて、『AI化候補トップ3』を見直してください。  
第02〜05回の座学を経て、『実は別の業務の方が向いている』と感じた方もいるかもしれません。今日が最後の変更チャンスです。  
取り組む業務課題を**1つ**に絞ってください。複数持っている方は、最も頻度が高く・負荷の大きいものを優先します。」

- __選定基準の提示（スライド）:__

| 優先すべき業務課題の条件 |
| --- |
| 週3回以上 or 月1回だが工数が大きい |
| 毎回同じような作業をしている |
| 「毎回これが面倒」とはっきり言える |
| 自分一人で完結する（他システム連携不要） |
| 機密情報を直接扱わなくてよい（or マスキングできる） |

---

### __【10\-25分】 ソリューション選定（Skills or Codex）__

- __フェーズ:__ 選定ワーク
- __スライド内容:__
	- __タイトル:__ 判断フローチャートで選ぼう

- __講師スクリプト案:__  
「判断フローチャートを使って、絞り込んだ業務課題にSkillsとCodexのどちらが向くかを確定します。わからない場合は講師に声をかけてください。個別に相談しながら決めます。」

- __判断フローチャート（再掲・板書も併用）:__

```
Q1. 解決したい業務は「ChatGPTとの会話の中で完結する」か？
  ├─ はい → Q2へ
  └─ いいえ（独立した画面・フォームが欲しい）→ 【Codex】

Q2. 毎回ほぼ同じ手順・判断を繰り返すか？
  ├─ はい → 【Skills】
  └─ いいえ（都度異なる入力・判断が必要）→ Q3へ

Q3. データの蓄積・一覧表示・集計が必要か？
  ├─ はい → 【Codex】
  └─ いいえ → 【Skills】
```

- __講師の巡回（全員の選定結果を確認）:__
	- Skills 組と Codex 組の人数をホワイトボードに書き出す
	- Codex を選んだが GitHub・Codex Web 連携が未完了の受講者には、今日の後半でフォロー

### __【25\-50分】 設計ワーク（Skills 組 / Codex 組に分かれて実施）__

- __フェーズ:__ 設計
- __スライド内容:__
	- __タイトル:__ 作るものを言葉で設計しよう（25分）

- __講師スクリプト案:__  
「Skills 組の方は**Skill 定義書**を、Codex 組の方は**アプリ仕様書**を記入してください。難しく考えず、『自分が何を作りたいか』を普通の言葉で書くだけです。これが第07回でAIに渡す指示の原型になります。」

---

#### __Skills 組：Skill 定義書の記入（25分）__

配布テンプレートに沿って以下を記入する：

1. **Skill 名：** （例：請求書送付依頼メール生成）
2. **どんな業務の自動化か：** （例：毎月末に取引先20社へ請求書送付依頼メールを送る）
3. **入力情報：** ユーザーが入力する内容（例：会社名・月・請求金額）
4. **出力：** ChatGPT が返すもの（例：件名付きの送付依頼メール本文）
5. **指示文（Instructions）の下書き：** 役割・条件・出力形式を日本語で書く
6. **テスト用の入力例：** 実際に試す入力を1つ用意する

---

#### __Codex 組：アプリ仕様書の記入（25分）__

配布テンプレートに沿って以下を記入する：

1. **アプリ名：** （例：配送依頼管理ツール）
2. **誰が使うか：** （例：業務担当 1〜3名）
3. **解決したい課題：** （例：毎日の配送依頼を紙で管理しているため、転記ミスや漏れが起きる）
4. **入力する情報（フォームの項目）：** （例：依頼日・配送先・品名・数量・担当者名）
5. **表示・管理したい情報：** （例：一覧表で全依頼を確認できる。完了/未完了で絞り込める）
6. **あると嬉しい機能（任意）：** （例：CSV でダウンロードできる）
7. **扱わない情報（機密データ等）：** （例：顧客の個人情報・金額は入力しない設計にする）

- __設計が詰まった受講者へのヒント（講師巡回時）:__
	- 「今、どんな手順でやっていますか？その手順をそのまま書いてください」
	- 「最終的にどんな画面が出たら『便利！』と感じますか？」
	- 「今使っているExcelのシートがあれば、それを見せてください。その列名がフォームの項目になります」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「Skills と Codex のどちらか決められません。両方やってもいいですか？」__
	- __A:__ 「第07〜09回の3回で1つを完成させる時間設計になっています。欲張らず、まず1つを完成させることを優先してください。研修後に2つ目に挑戦することを強くおすすめします。」

- __Q2: 「私のやりたいことは、既存の基幹システムと連携が必要です」__
	- __A:__ 「既存システムとの直接連携は今回の研修スコープ外です。まず『既存システムからCSVでデータを出力して、Codexで作ったアプリに取り込む』という間接連携を設計しましょう。多くの業務課題はこの形で解決できます。」

- __Q3: 「アプリ仕様書を書いたことがないので不安です」__
	- __A:__ 「難しく考えなくて大丈夫です。Codexに渡す指示文の下書きを作っているだけです。完璧でなくても、第07回でCodexに入力しながら一緒に育てていきます。今日は思ったことを書いておくだけでOKです。」
