# Skill: NotebookLM スライド生成プロンプト作成

## 概要

セミナーの講師用ピッチ資料（Markdownファイル）を元に、**NotebookLM の「スライド資料」機能**で高品質なスライドを生成するためのプロンプトを出力する。

ブランドカラーと背景画像を固定することで、全回を通じて統一されたデザインのスライドを生成できる。

## 前提

- ブランドカラー定義: `materials/brand-colors.md`
- ビジュアルスタイルガイド: `materials/visual-style-guide.md`（スライド型定義・視覚的デザインルール）
- 背景画像: `materials/background.png`（`materials/background.svg` から生成。再生成は `cd materials && npm run render-png`）
- リファレンス画像: `materials/reference-slides/`（型ごとの見本スライド画像）
- 入力: 各回の `第○○回_ピッチ資料（講師用スライド構成）.md`

## 使い方

1. Cursor で `/seminar-slide-generator` を実行
2. 「第○○回のスライド用プロンプトを生成して」と指示
3. 出力されたプロンプト（Markdown）をコピー
4. NotebookLM で新しいノートブックを作成
5. ソースとして以下をアップロード:
   - `materials/brand-colors.md`
   - `materials/visual-style-guide.md`
   - `materials/background.png`
   - `materials/reference-slides/` 内のリファレンス画像（あれば）
6. 「スライド資料」ボタンを押し、コピーしたプロンプトを貼り付けて生成

## 処理手順

### Step 1: ピッチ資料の読み取り

指定された回のピッチ資料を読み込み、以下を抽出する:

- 全体構成サマリー（セクション・スライド数・時間配分）
- 各スライドの内容（タイトル・本文・表・デザイン提案）
- スライド総数

### Step 2: ブランドカラー・ビジュアルスタイルの参照

`materials/brand-colors.md` からカラーコードを読み取る:

| トークン | HEX | 用途 |
|----------|-----|------|
| Primary | #005BAC | 見出し背景・プログレスバー・ステップ番号 |
| Secondary | #02396B | ヘッダー・フッター背景 |
| Accent | #F58220 | CTA・ハイライト・重要メッセージ枠 |
| Light Gray | #F5F5F5 | スライド背景 |

`materials/visual-style-guide.md` からスライド型定義を読み取り、各スライドに適切な型（型A〜型L）を割り当てる:

| 型 | 名前 | 主な用途 |
|----|------|----------|
| 型A | タイトルスライド | 各回の冒頭1枚目 |
| 型B | カード横並び型 | 振り返り、特徴紹介、比較 |
| 型C | テーブル/比較表型 | 機能比較、まとめ表 |
| 型D | タイムライン/ステップ型 | アジェンダ、操作手順 |
| 型E | Before/After型 | 導入前後の比較 |
| 型F | ワーク/タイマー型 | ハンズオン作業中 |
| 型G | チェックリスト型 | 到達目標、確認ポイント |
| 型H | Q&A型 | よくある質問 |
| 型I | フローチャート型 | 判断フロー |
| 型J | アンケート/QR型 | フィードバック |
| 型K | コンテンツスライド型 | 概念説明、デモなど汎用 |
| 型L | 配布資料型 | 印刷配布物 |

### Step 3: プロンプトの生成

以下のテンプレートに従い、NotebookLM に貼り付ける用のプロンプトを出力する。

---

## 出力テンプレート

以下を Markdown テキストとして出力する（NotebookLM の「追加の指示」欄に貼り付ける用）:

```
## スライド仕様

- スライド枚数: {ピッチ資料から取得した総数}枚
- 言語: 日本語
- 1スライド1メッセージ（テキストは最大5〜6行）
- 全スライドにプログレスバー（セクション名 + スライドNo.）を固定ヘッダーとして表示

## ブランドカラー（厳守）

以下のカラーのみを使用してください。他の色は使わないでください。

- メインカラー（青）: #005BAC → 見出し背景、プログレスバー、ステップ番号の丸
- サブカラー（濃紺）: #02396B → ヘッダー・フッター背景
- アクセントカラー（オレンジ）: #F58220 → CTA、ハイライト、重要メッセージ枠
- ベースカラー: #F5F5F5 → スライド背景
- テキスト（濃色背景上）: #FFFFFF
- テキスト（明色背景上）: #171717

## ビジュアルスタイル（厳守）

添付の visual-style-guide.md に定義されたデザインルールに従ってください。主なルール:

- クリーンでモダンなフラットデザイン。過度な装飾・グラデーション・3D表現は使わない
- アイコンはシンプルなライン（線画）スタイルで統一
- カード要素には薄いドロップシャドウ（blur 8〜12px, opacity 10〜15%）を適用
- カード・ボタン・枠の角丸は 8〜12px で統一
- スライド四辺に十分なマージン（上下左右 5% 以上）を確保

各スライドには「スライド型」（型A〜型L）を指定しています。各型のレイアウトルールは visual-style-guide.md を参照してください。

## リファレンス画像

添付のリファレンス画像（ref-*.png）のレイアウトとビジュアルスタイルを踏襲してください。配色・余白・カード形状・影の強さ・フォントの雰囲気を合わせてください。

## 背景画像

- 添付の background.png を全スライドの背景として使用してください
- 背景はスライドごとに変えず、全スライドで統一してください

## ロゴスペース

- 右上（約 120×40px 相当）にロゴ配置用のスペースを確保してください
- このスペースにはコンテンツを配置しないでください
- 白い矩形で囲わず、背景に馴染ませてください

## フォント

- 見出し: Noto Sans JP Bold（36〜48pt相当）
- 本文: Noto Sans JP Regular（18〜22pt相当）

## トーン

- DX研究所のブランドらしく、プロフェッショナルかつ親しみやすい
- 専門用語は噛み砕いた補足を添える
- 物流事務の現場担当者が直感的に理解できる表現

## スライド構成

{ここに各スライドの構成を展開。各スライドに必ず「スライド型」と「デザイン」を記載する}

### スライド 1: {タイトル}
- スライド型: 型A（タイトルスライド）
- 大見出し: {内容}
- サブテキスト: {内容}
- デザイン: {デザイン提案}

### スライド 2: {タイトル}
- スライド型: {型B〜型L のいずれか}
- 内容: {内容}
- デザイン: {デザイン提案}
...

（以下、全スライド分を展開）
```

## 注意事項

- NotebookLM の「プレゼンテーションの長さ」は「短め」または「デフォルト」を推奨
- 1回の生成で全スライドを作ることを目指す（再生成すると背景が変わる可能性がある）
- 生成後のスライドは全て画像形式。PowerPoint にインポートしてからロゴを手動配置する
- 講師メモ（ノート欄）の内容はプロンプトに含めない（NotebookLM のスライドにはノート機能がないため）
- **全スライドに「スライド型」と「デザイン」の指示を必ず含めること。** デザイン指示がないスライドがあると、NotebookLM が自由解釈しトンマナがずれる原因になる
- **各回のプロンプトで visual-style-guide.md への参照を必ず含めること。** これにより回をまたいだデザインの一貫性が保たれる
