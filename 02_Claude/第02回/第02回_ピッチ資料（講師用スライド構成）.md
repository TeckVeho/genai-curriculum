#### Slide 08｜チャット型 → エージェント型の進化

**タイトル：** AI活用の進化 ── 「忙しくなる」から「楽になる」へ

**成熟度の2段階（大きなフロー図）：**

```
【段階1：チャット型】
  指示 → AIが返答 → 人間がコピペ・転記・手作業
  → できることは増えたが、人間は忙しいまま

        ↓ 進化 ↓

【段階2：エージェント型（Cowork）】
  指示 → AIが自動実行 → 完成品がそのまま手に入る
  → 人間は承認・確認するだけ → 楽になる
```

**受講者への問いかけ：**
> 「ChatGPTを使い始めて、**できることは増えたけど楽になった実感はない**…という方はいませんか？」

**キーメッセージ：**
> チャット型の限界を超えるのが「エージェント型」。Coworkはまさにこの段階。

> **講師メモ（ノート欄）：**  
> 「ChatGPTを使ってキャパは2〜3倍に広がったけど、楽になったとは言えない──これはよくある感覚です。それはチャット型の限界で、結局人間がコピペや転記をする必要があるから。Cowork＝エージェント型は、そこを自動化します。成熟度が上がるほど、人間は楽になる。今日のデモでその違いを体感してください。」

#### Slide 09｜Coworkの操作サイクル

**タイトル：** Coworkの基本サイクル ── 5ステップ

**サイクル図（円形フロー）：**

```
① 指示 → ② 計画 → ③ 承認 → ④ 実行 → ⑤ 確認
   ↑                                          |
   └──────── フィードバック ←──────────┘
```

**各ステップの説明：**

| ステップ | やること | 誰が？ |
|---|---|---|
| ① 指示 | やりたいことを日本語で伝える | **人間** |
| ② 計画 | Claudeが作業計画を提示する | **Claude** |
| ③ 承認 | 計画を確認して「OK」を出す | **人間** |
| ④ 実行 | Claudeがタスクを自律的に実行する | **Claude** |
| ⑤ 確認 | 成果物を確認し、必要なら修正指示を出す | **人間** |

**キーメッセージ：**
> Coworkは「丸投げ」ではない。**人間が承認・確認するステップ**が必ず入る ＝ 安全

> **講師メモ（ノート欄）：**  
> 「Coworkは放任ではなく委任です。③の承認ステップで計画を見てからOKを出す、⑤の確認ステップで成果物をチェックする。人間が関与するポイントが設計されているから、安全に使えます。このサイクルは第03回以降で何度も体験します。」

#### Slide 10｜講師デモ：Coworkでフォルダを自動整理

**タイトル：** 【デモ】散らかったフォルダをCoworkで一瞬で整理する

**デモシナリオ：**
> 命名規則バラバラの20個のファイル（請求書・納品書・メモ・画像）が入った「未整理」フォルダを、Coworkに指示して自動整理する

**事前準備（デモ用フォルダ）：**
> デスクトップに「未整理」フォルダを作成し、以下のようなファイルを20個程度入れておく

```
未整理/
  請求書_A社.xlsx          ← 命名バラバラ
  202606納品書B社.pdf
  メモ.txt
  IMG_4821.jpg             ← 現場写真
  配送報告0612.docx
  seikyu_C社_2026.xlsx
  スクリーンショット 2026-06-15.png
  ...（計20ファイル程度）
```

**デモの流れ（4ステップ）：**

| # | 操作 | 見せるポイント |
|---|---|---|
| ① | Coworkを開き、デスクトップの「未整理」フォルダを指定する | 作業対象フォルダの設定 |
| ② | 「このフォルダのファイルを種類別にサブフォルダに分けて、ファイル名を『YYYYMMDD_種別_取引先名』の形式に統一して」と指示する | 日本語だけで指示できること |
| ③ | Claudeが計画を提示（どのファイルをどう分類するか）→ 講師が内容を確認して承認 | **承認ステップ**で安全を確認できること |
| ④ | 実行後、Finderで整理されたフォルダを開いて見せる | **ファイルが実際に移動・リネームされている** |

**整理後のフォルダ（Finderで見せる）：**
```
未整理/
  請求書/
    20260601_請求書_A社.xlsx
    20260615_請求書_C社.xlsx
  納品書/
    20260606_納品書_B社.pdf
  報告書/
    20260612_配送報告.docx
  写真/
    20260610_現場写真.jpg
    20260615_スクリーンショット.png
  メモ/
    メモ.txt
```

**デモ後に強調するポイント：**
- コードは一行も書いていない
- ファイルを**作った**のではなく**動かした・リネームした** → ChatGPTには絶対できない
- 20ファイルの整理を手作業でやったら何分かかるか？ → **Coworkなら1〜2分**
- 計画を確認してからOKを出した（安全なサイクル）

> **講師メモ（ノート欄）：**  
> Coworkを画面共有で操作する。最重要ポイントは「ファイルが移動・リネームされる」こと。ChatGPTはテキストを返すだけだが、Coworkは**パソコン上のファイルを直接操作する**。この違いを一目で体感させる。デモ前にFinderで「未整理」フォルダの中身を見せ（Before）、実行後にもう一度開いて整理された状態を見せる（After）。「20ファイルの整理、手作業なら15〜20分。Coworkなら1分です」と時間差を強調する。

**タイトル：** スキル ── AI向けの「業務マニュアル」

**スキルの定義（大きなフォント）：**
> スキル ＝ 再現性の高いAIワークフローを定義した **.mdファイル**

**比較で理解する：**

| 比較対象 | 対象 | 内容 |
|---|---|---|
| 業務マニュアル | 人間向け | 「この手順でやってください」 |
| **スキル** | **AI向け** | **「この手順でやってください」（日本語で書ける）** |

**スキルがあると何が変わるか（Before / After）：**

| スキルなし | スキルあり |
|---|---|
| 毎回「こういうフォーマットで、こういう手順で…」と長い指示を書く | スラッシュコマンド1つで**同じ品質のアウトプット**が毎回出る |
| 指示の書き方次第で品質がブレる | フォーマット・手順・品質が**固定**される |
| 他の人に共有できない | ファイルを渡せば**誰でも同じことができる** |

> **講師メモ（ノート欄）：**  
> 「スキルは人間向けの業務マニュアルと同じ発想です。新人に『このマニュアル通りにやって』と渡すように、AIに『このスキル通りにやって』と渡します。しかも日本語で書けます。プログラミングではありません。第03回からこのスキルを実際に使い、第06回からは自分で作れるようになります。」

#### Slide 14｜プラグインの構造

**タイトル：** プラグインの構造 ── スキルの入れ物

**階層図（大きく表示）：**

```
個人プラグイン（1つのまとまり）
  ├── skills/          ← スキル群（.md ファイル）
  │     ├── skill1.md    例：セミナー資料作成
  │     ├── skill2.md    例：フォルダ整理
  │     └── skill3.md    例：レポート生成
  ├── connectors/      ← コネクター（外部アプリ連携・上級）
  └── README.md        ← プラグインの説明
```

**スキル vs コマンドの違い：**

| 種類 | 動作 | 使い方 |
|---|---|---|
| **スキル** | AIが状況を判断して**自動的に使う** | 「これ使えばいいな」とAIが判断 |
| **コマンド** | スラッシュコマンドを**明示的に入力**して実行 | `/フォルダ整理` のように手動で呼び出す |

**キーメッセージ：**
> プラグイン ＝ スキルをまとめた「道具箱」。第03回から実際にスキルを使い始めます

> **講師メモ（ノート欄）：**  
> 「プラグインはスキルの入れ物です。1つのプラグインの中に複数のスキルが入っています。また、スキルはAIが勝手に使ってくれるモード、コマンドはスラッシュで明示的に呼び出すモード。どちらも設定できます。構造を知っておくと、第06回以降で自分のスキルを作るときに迷いません。」

#### Slide 15｜デモ：スラッシュコマンドでスキルを発動

**タイトル：** 【デモ】スラッシュコマンドでスキルを動かす

**デモシナリオ：**
> 講師が実際にスラッシュコマンドを入力し、スキルが発動する様子を見せる

**デモの流れ：**

| # | 操作 | 見せるポイント |
|---|---|---|
| ① | Coworkでスラッシュ `/` を入力 → コマンド一覧が表示される | スキルが一覧で見えること |
| ② | コマンドを選択し、簡単な指示を追加する | 「日本語の一言で動く」こと |
| ③ | スキルが起動 → 右側にコンテキスト情報が表示される | 事前定義のルールが読み込まれること |
| ④ | AIが質問→回答→実行→成果物が保存される | 対話しながらタスクが進むこと |

**デモ後に強調するポイント：**
- スキルがあれば**毎回同じ品質**で出力される
- スキルは**.mdファイル**（日本語テキスト）でできている
- 第03回から皆さんも同じことをやります

> **講師メモ（ノート欄）：**
