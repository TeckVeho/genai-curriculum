# __第07回：[ワークショップ] ソリューション構築 (1)：プロトタイプ開発__

---

## __1\. 講義概要__

- __フェーズ:__ 実践
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第06回で作成した設計書をもとに、**実際に動くプロトタイプを作る**。Cowork タスクの最小構成（入力 → 処理 → 成果物出力）を完成させることがゴール。「完璧にしようとせず、まず動かす」という姿勢を定着させる。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. 設計書を Cowork への指示文として変換し、タスクを投入できる
2. プロトタイプ（最小動作版）を1つ完成させられる
3. 「思い通りに動かないとき」に修正指示を出せる
4. 次回（第08回）で改善すべき点を3つ以上書き出せる

### __準備物（講師）__

- PC（Claude Desktop の Cowork が動作する状態）
- 第06回で各受講者が作成した設計書（講師もコピーを持っておくと巡回指導がしやすい）
- トラブルシューティング参考資料（Cowork のよくあるエラーと対処法）

### __準備物（受講者）__

- PC（Claude Desktop が利用可能な状態）
- 第06回で作成した Cowork タスク設計書

### __【05\-50分】 構築ワーク__

- __フェーズ:__ 個人作業（講師は全員を巡回）

---

#### __全員共通：Cowork プロトタイプの構築__

**Step 1：設計書を Cowork への指示文に変換する（10分）**

設計書の「Claude への指示文の下書き」を、実際に Cowork に入力できる形に仕上げる。以下のテンプレートを参考に整理する：

```
[タスク名] を実行してください。

■ 入力データ：
- [データの場所とファイル名、または手入力の条件]

■ 期待する成果物：
- [成果物①のファイル名と仕様]
- [成果物②のファイル名と仕様]（あれば）

■ 成果物の保存先：
- デスクトップの「研修出力」フォルダ

■ 条件・注意点：
- [文体・文字数・書式等の条件]
- [含めるべき要素・含めてはいけない情報]
```

**Step 2：Cowork にタスクを投入する（15分）**

1. Claude Desktop の Cowork タブを開く
2. 作成した指示文を入力して送信
3. Claude が計画を立てる → 内容を確認して承認
4. 実行過程を観察しながら待つ
5. 完了後、成果物ファイルを開いて確認

**Step 3：動作確認と修正指示（10分）**

成果物を開き、以下を確認する：

- [ ] 期待した形式のファイルが生成されている
- [ ] 内容（データ・文章・書式）が設計書の仕様に沿っている
- [ ] 関数やグラフがある場合、正しく動作している

問題があれば修正指示を出す（例：「集計の合計行が抜けています。追加してください」「文体がカジュアルすぎます。ビジネス文体に変えてください」）

**Step 4：プロジェクト化する（余裕があれば）**

繰り返し使うタスクの場合、Cowork のプロジェクトとして保存する：
1. サイドバーから「新しいプロジェクト」を作成
2. プロジェクト名・指示を設定
3. 関連ファイルをプロジェクトに追加
4. プロジェクト内でタスクを再実行して動作確認

---

- __講師の巡回ポイント:__
	- 「タスクが動かない」→ 指示文が曖昧でないか確認。具体的なファイル名・形式・条件を追加するよう促す
	- 「成果物の内容がおかしい」→ 条件を1つずつ確認し、足りていない制約を追加
	- 「何から手をつければいいかわからない」→ 「まず設計書の一番シンプルな部分だけで指示を出しましょう。成果物が1つできることを確認してから次に進みましょう」
	- 「ファイルが見つからない」→ 保存先のフォルダパスを確認。Claude に「ファイルをどこに保存しましたか？」と聞く

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「Cowork がエラーを返してきました。どうすればいいですか？」__
	- __A:__ 「エラーの内容をそのまま Claude に伝えて『この問題を解決してください』と指示してください。Claude は自分が出したエラーを自分で直すのが得意です。」

- __Q2: 「プロトタイプが完成しませんでした」__
	- __A:__ 「大丈夫です。今日完成しなかった分は次回の冒頭15分で追いつく時間を設けます。今日の作業内容と、どこで詰まったかをメモしておいてください。」

- __Q3: 「Cowork の出力が毎回少し違います。安定させるにはどうすればいいですか？」__
	- __A:__ 「指示文に出力の構造を具体的に書くと安定します。例：『必ず以下の構成で出力してください：①件名 ②本文（3段落） ③締めの一文』のように明示してください。また、プロジェクトの指示に入れておくと毎回適用されます。」
