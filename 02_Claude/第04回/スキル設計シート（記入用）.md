## 記入例 ── 月次配送実績レポート作成スキル

**スキル名：** 月次配送実績レポート作成スキル

**記入日：** 2026年7月1日　**記入者：** 山田太郎（物流事務担当）

---

### Step 1：Claudeに頼んだこと

**Claudeに何を頼んだか：**

```
毎月やっている配送実績の集計作業を手伝ってほしい。
基幹システムから出力したCSVファイルを集計して、上司に報告するレポートを作って。
```

**渡したデータ・ファイル：**

```
2026年4月の配送実績CSV（10件分）
```

---

### Step 2：結果を見て整理する

#### OKだった部分 / NGだった部分

| | Claudeがやったこと | OK / NG | NGの場合、正しくはどうすべきか |
|---|---|---|---|
| 1 | CSVを読み込んだ | ✅ OK | |
| 2 | 全件をまとめて件数・重量を集計した | ❌ NG | **配送先コード別**に集計すべき |
| 3 | テキスト形式で結果を表示した | ❌ NG | **Excel形式**（グラフ付き）で出力すべき |
| 4 | サマリー文面を生成した | ✅ OK | |
| 5 | 遅延に関する記載なし | ❌ NG | 遅延率5%超は「要注意」と記載すべき |
| 6 | — | — | |

**気づき：** 「配送先別に分ける」「Excelグラフで見せる」「遅延率をチェックする」は自分が無意識にやっていた判断だった

---

#### 設計シート 12項目への転記

| No. | 項目 | 記入例 |
|---|---|---|
| ① | **目的** | 毎月の配送実績データを自動集計し、上司報告用のサマリーレポートを生成する。手作業での集計ミスを防ぎ、作業時間を3時間→15分に短縮する |
| ② | **利用者** | 物流事務担当 1名（山田）。毎月月初に使用 |
| ③ | **起動条件** | 毎月1日に基幹システムから前月の配送実績CSVが出力されたとき、手動でスキルを起動する |
| ④ | **入力データ** | ①配送実績CSV（基幹システム出力・列：配送日,配送先コード,重量kg,出荷日,到着日） ②顧客マスタExcel（列：配送先コード,配送先名,地域,担当者） |

| No. | 項目 | 記入例 |
|---|---|---|
| ⑤ | **処理手順** | 下記の詳細欄参照 |
| ⑥ | **判断基準** | ★ 遅延率（到着日−出荷日が3日超）が5%を超える配送先 → サマリーに「要注意」マークを付与。★ 前月比20%以上の件数増減がある配送先 → サマリーに「変動大」と記載 |
| ⑦ | **出力形式** | ★ Excel形式（シート1：集計表、シート2：棒グラフ） ＋ 上司向けサマリーメール文面（テキスト、3〜5行、です/ます調） |
| ⑧ | **完了条件** | ①全配送先コードが顧客マスタと突合済み ②集計Excelのグラフが正しく表示 ③サマリーメール文面が生成済み ④出力フォルダにファイルが2つ保存済み |

> ★ マーク = Claudeの結果を見て気づいた修正点

**⑤ 処理手順の詳細：**

```
1. 配送実績CSVを読み込み、ヘッダー行を確認する
2. ★ 配送先コードをキーに顧客マスタExcelと突合し、配送先名を付与する
3. ★ 配送先別に以下を集計する：配送件数、総重量(kg)、平均リードタイム(日)
4. ★ 遅延率（リードタイム3日超の割合）を配送先別に算出する
5. ★ 集計結果を所定フォーマットのExcelシート1に転記する
6. ★ シート2に配送先別件数の棒グラフを作成する
7. 集計結果をもとに、上司向けサマリーメール文面（3〜5行）を生成する
8. 出力ファイルを指定フォルダに保存する
```

> ★ マーク = Claudeは「全件まとめて集計 → テキスト表示」だったのを修正

---

### Step 3：安全設計

| No. | 項目 | 記入例 |
|---|---|---|
| ⑨ | **例外処理** | CSVのヘッダーが想定と異なる場合 → 処理を停止し「フォーマット不一致：列名〇〇が見つかりません」と報告する。顧客マスタに存在しない配送先コードがある場合 → 「不明」として集計に含め、サマリーに「未登録コードあり」と注記する |
| ⑩ | **人間の確認箇所** | 集計結果の合計値が先月と大幅に乖離していないか目視確認してからメール送付する。「要注意」マーク付きの配送先がある場合、原因を確認してからレポートを提出する |
| ⑪ | **禁止事項** | 顧客の個人情報（担当者の電話番号・メールアドレス）を出力に含めない。取引金額情報は一切扱わない。メールを自動送信しない（文面生成のみ）。元データ（CSV・マスタ）を変更・削除しない |
