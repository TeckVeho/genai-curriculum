#### Slide 04｜「スキル単体」と「AI業務担当者」の違い

**タイトル：** スキルだけでは足りない ── AI業務担当者への進化

**比較テーブル（大きく表示）：**

| | スキル単体（Phase①で作ったもの） | AI業務担当者（Phase②で作るもの） |
|---|---|---|
| **できること** | 1つの作業手順を再現 | 業務全体を担当 |
| **知識** | 指示文に書いた内容だけ | 業務の背景・ルール・過去データも参照 |
| **実行タイミング** | 人間が毎回起動 | スケジュールで自動実行も可能 |
| **判断の範囲** | 指示通りに動く | 参照情報をもとに適切に対応 |
| **人間の関わり** | 毎回指示を出す | 承認ポイントで確認、それ以外は任せる |

**キーメッセージ：**
> スキル＝「手順書」。AI業務担当者＝「手順書＋業務知識＋スケジュール＋責任範囲」をまとめた仕組み。

> 💡 **講師メモ（ノート欄）：**  
> 「Phase①で作ったスキルは『手順書』です。毎回自分で起動して、毎回同じ指示を出していましたよね。AI業務担当者は、そのスキルに『この業務の背景知識』『いつ実行するか』『どこまでAIに任せてよいか』を加えたものです。人間でいえば、新人に業務マニュアルだけ渡すのと、業務の背景や判断基準まで伝えて任せるのとの違いです。」

**構成図（中央に大きく）：**


> 💡 **講師メモ（ノート欄）：**

#### Slide 10｜デモ2：スケジュールタスクで定期実行を設定する

**タイトル：** デモ2：毎週月曜に自動実行される設定

**画面構成（デモ画面の説明）：**

**操作手順（講師が実演）：**
1. プロジェクト内で「スケジュールタスク」を新規作成
2. 実行タイミングを設定：「毎週月曜日 9:00」
3. 実行する内容を指定：「配送実績CSVを読み込み、前週の配送先別集計レポートを生成する」
4. 設定を保存 → スケジュール一覧に表示されることを確認

**ビジュアル（スケジュール設定画面のモックアップ）：**

```
┌─────────────────────────────────┐
│  スケジュールタスク               │
│                                 │
│  実行：毎週月曜日 9:00           │
│  内容：配送実績レポート生成       │
│  ステータス：有効                │
│                                 │
│  次回実行予定：2026/7/28 9:00    │
└─────────────────────────────────┘
```

**キーメッセージ：**
> 一度設定すれば、人間が毎回「実行して」と指示しなくてよい。  
> ただし **出力結果は人間が確認してから社内に共有する** ルールで運用する。

> 💡 **講師メモ（ノート欄）：**  
> 「スケジュールタスクを設定しました。これで毎週月曜の朝にAIがレポートを自動生成します。ただし、生成されたレポートをそのまま上司に送るのではなく、まず自分が内容を確認してからメールで送る、という運用にします。ここが『承認ポイント』です。AIが全自動でメール送信まですることはしません。」

#### Slide 11｜デモ3：参照情報を追加して出力品質を上げる

**タイトル：** デモ3：業務知識を登録すると、AIの出力がこう変わる

**操作手順（講師が実演）：**

2. 以下の業務知識を追加登録：

   - 「遅延率5%以上はアラート対象」
   - 「レポートのフォーマットは社内テンプレートに準拠」

**キーメッセージ：**
> 同じスキル（手順）でも、**参照情報（業務知識）があるだけで出力品質が段違い**に上がる。

> 💡 **講師メモ（ノート欄）：**

> A：スケジュール実行の結果は通知されます。エラーが起きた場合は実行が停止し、人間が対処方法を判断します。**勝手にリトライして暴走することはありません。**

**Q4：「参照情報に何を書けばいいか分からないのですが…」**
> A：次回（第07回）の設計回で詳しくやります。今日は「業務知識を追加すると出力が良くなる」ことだけ体験できればOKです。
