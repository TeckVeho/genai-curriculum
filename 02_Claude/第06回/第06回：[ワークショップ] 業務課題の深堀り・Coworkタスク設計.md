# __第06回：[ワークショップ] 業務課題の深堀り・Coworkタスク設計__

---

## __1\. 講義概要__

- __フェーズ:__ 実践（ワークショップ開始）
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第01回のワークフロー棚卸しに立ち返り、座学（第02〜05回）で得た知識を使って**「自分の課題に最適な Cowork ソリューションを自分で設計する」**。この回で決めた設計が第07〜09回の構築の土台になるため、「作るものが明確になった状態」で終わることが目標。各自の設計内容に講師が個別フィードバックする時間も設ける。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. 第01回のワークフロー棚卸しシートを再評価し、取り組む業務課題を1つに絞れる
2. Cowork ソリューション設計フローチャートを使い、成果物の種類・定期実行の有無・プロジェクト化の要否を確定できる
3. Cowork タスク設計書（指示文の原型）を自然言語で書き出せる
4. 第07回以降で構築を開始できるレベルまで設計を具体化できる

### __準備物（講師）__

- ホワイトボード・マーカー（設計方針の板書用）
- 配布資料：**Cowork タスク設計書テンプレート**（本テキスト4節参照）
- 第01回のワークフロー棚卸しシート（受講者が持参するが、予備を印刷しておく）
- Cowork ソリューション設計フローチャート（再配布用）
- 付箋・ペン（課題の書き出しワーク用）

### __準備物（受講者）__

- PC（Claude Desktop が利用可能な状態）
- 第01回のワークフロー棚卸しシート
- 第05回の Cowork ソリューション設計フローチャート（仮の答えを記入済み）

---

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

---

### __【00\-10分】 棚卸しシートの再確認と課題の絞り込み__

- __フェーズ:__ 振り返り・絞り込み
- __スライド内容:__
	- __タイトル:__ 第01回のワークフロー棚卸しシートを再評価しよう

- __講師スクリプト案:__  
「ここからの4回は**自分の業務ワークフローを自動化するソリューションを実際に作る**時間です。まず第01回のワークフロー棚卸しシートを開いて、『自動化したいワークフロー トップ3』を見直してください。  
第02〜05回の座学を経て、『実は別のワークフローの方が向いている』と感じた方もいるかもしれません。今日が最後の変更チャンスです。選定基準は第01回と同じ3条件（繰り返し頻度が高い＋手順が毎回同じ＋入出力パターンが明確）です。  
取り組む業務課題を**1つ**に絞ってください。複数持っている方は、最も頻度が高く・負荷の大きいものを優先します。」

- __選定基準の提示（スライド）:__

| 優先すべき業務課題の条件 |
| --- |
| 週3回以上 or 月1回だが工数が大きい |
| 毎回同じような作業をしている |
| 「毎回これが面倒」とはっきり言える |
| 自分一人で完結する（他システム連携不要） |
| 機密情報を直接扱わなくてよい（or マスキングできる） |

### __【10\-20分】 ソリューション方針の確定__

- __フェーズ:__ 方針決定ワーク
- __スライド内容:__
	- __タイトル:__ 設計フローチャートで方針を決めよう

- __講師スクリプト案:__  
「設計フローチャートを使って、絞り込んだ業務課題にどのパターンが向くかを確定します。わからない場合は講師に声をかけてください。個別に相談しながら決めます。」

- __設計フローチャート（再掲・板書も併用）:__

```
Q1. 解決したい業務の成果物は何か？
  ├─ 文書・メール・報告書 → 【文書生成タスク】
  ├─ Excel・集計表・グラフ → 【データ処理タスク】
  ├─ ファイルの整理・変換・リネーム → 【ファイル管理タスク】
  └─ 上記の組み合わせ → 【複合タスク】

Q2. その作業は定期的に繰り返すか？
  ├─ はい → 【スケジュールタスク】に設定
  └─ いいえ → 都度実行

Q3. 同じ前提条件を毎回使うか？
  ├─ はい → 【プロジェクト】にまとめる
  └─ いいえ → 単発タスクで十分
```

- __講師の巡回（全員の方針を確認）:__
	- 各受講者の成果物タイプ・定期実行の有無・プロジェクト化の要否をホワイトボードに書き出す
	- 方針が決まらない受講者には「第03〜05回のどのハンズオンが一番自分の業務に近かったですか？」と問いかける

#### __Cowork タスク設計書の記入（30分）__

配布テンプレートに沿って以下を記入する：

1. **タスク名：** （例：月次配送実績レポート自動生成）
2. **解決したい課題：** （例：毎月の配送実績を手作業でExcelにまとめている。集計ミスが起きやすく、2〜3時間かかる）
3. **誰が使うか：** （例：事務担当 1名）
4. **入力データ：** （例：配送実績CSV + 顧客マスタExcel）
5. **期待する成果物：** （例：配送先別集計Excel（グラフ付き） + 上司向けサマリーメール文面）
6. **成果物の形式：** □文書（txt/Word） □Excel □PowerPoint □CSV □その他（　　）
7. **繰り返し頻度：** □毎日 □毎週 □毎月 □不定期
8. **プロジェクト化：** □する（同じ前提条件を毎回使う） □しない（都度指示で十分）
9. **Claude への指示文の下書き：** 実際に Cowork に入力する指示文を、日本語でできるだけ具体的に書く
10. **テスト用の入力例：** 実際に試す入力データまたは条件を1つ用意する
11. **扱わない情報（機密データ等）：** （例：顧客の個人情報・金額は入力しない設計にする）

- __設計が詰まった受講者へのヒント（講師巡回時）:__
	- 「今、どんな手順でやっていますか？その手順をそのまま書いてください」
	- 「最終的にどんなファイルが手元にあったら『便利！』と感じますか？」
	- 「今使っているExcelのシートがあれば、それを見せてください。その列名がそのまま指示文に使えます」
	- 「第03〜05回のハンズオンで出した指示文を参考にしてください。あれを自分の業務に置き換えるだけです」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「複数のタスクを作りたいのですが、いいですか？」__
	- __A:__ 「第07〜09回の3回で1つを完成させる時間設計になっています。欲張らず、まず1つを完成させることを優先してください。研修後に2つ目に挑戦することを強くおすすめします。」

- __Q2: 「私のやりたいことは、既存の基幹システムと連携が必要です」__
	- __A:__ 「既存システムとの直接連携は今回の研修スコープ外です。まず『既存システムからCSVでデータを出力して、Cowork に読み込ませて加工する』という間接連携を設計しましょう。多くの業務課題はこの形で解決できます。」

- __Q3: 「設計書を書いたことがないので不安です」__
	- __A:__ 「難しく考えなくて大丈夫です。Cowork に渡す指示文の下書きを作っているだけです。完璧でなくても、第07回で Claude に入力しながら一緒に育てていきます。今日は思ったことを書いておくだけでOKです。」
