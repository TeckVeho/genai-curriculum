## 設計フローチャート

```
Q1. 解決したい業務の成果物は何か？
  ├─ 文書・メール・報告書 → 【文書生成タスク】
  ├─ Excel・集計表・グラフ → 【データ処理タスク】
  ├─ ファイルの整理・変換・リネーム → 【ファイル管理タスク】
  └─ 上記の組み合わせ → 【複合タスク】

Q2. その作業は定期的に繰り返すか？
  ├─ はい（毎日/毎週/毎月） → 【スケジュールタスク】に設定
  └─ いいえ（必要なときだけ） → 都度実行

Q3. 入力データはどこにあるか？
  ├─ ローカルファイル（Excel・CSV等） → フォルダを指定して読み込ませる
  ├─ Webから取得する情報 → Web検索と組み合わせる
  └─ 毎回手入力する → タスク指示文にテンプレートを含める

Q4. 同じ前提条件を毎回使うか？
  ├─ はい → 【プロジェクト】にまとめる
  └─ いいえ → 単発タスクで十分
```

---

## 4つの活用パターン早見表

| パターン | 具体例 | 向いている業務 |
|---------|-------|--------------|
| **文書生成** | メール・報告書・マニュアルをファイルとして自動作成 | 定型メール、議事録、手順書 |
| **データ処理** | Excel作成・集計・グラフ生成 | 月次報告、実績集計、売上分析 |
| **ファイル管理** | フォルダ整理・リネーム・変換 | ダウンロードフォルダ整理、ファイル名統一 |
| **複合タスク** | 上記の組み合わせを1回の指示で実行 | データ集計＋レポート作成 |
