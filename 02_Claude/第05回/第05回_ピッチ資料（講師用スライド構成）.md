#### Slide 05｜プロジェクト ── デモ

**タイトル：** デモ：プロジェクトを作ってみる

**デモ操作手順：**
1. Cowork のサイドバーで「プロジェクト」→「新しいプロジェクトを作成」
2. プロジェクト名：「月次配送レポート」
3. プロジェクト指示を設定：

```
このプロジェクトは月次の配送実績レポート作成に使います。
- 入力：配送実績のCSVファイル（列：日付・配送先・品名・数量・単価・金額）
- 出力：集計Excel（配送先別集計＋グラフ）＋上司向けサマリーメール文面
- 文体：ビジネス文体、簡潔に
```

4. プロジェクトにファイル（前回の CSV）を追加
5. プロジェクト内でタスクを実行：「今月分のレポートを作成してください」
6. プロジェクトの指示が自動的に参照されることを確認

**ポイント（大きく表示）：**
> 「プロジェクト指示を1回設定すれば、毎回同じ前提条件を書く必要がありません」

> 💡 **講師メモ（ノート欄）：**  
> 実際にプロジェクトを作成し、指示を設定して、タスクを実行する過程を見せる。「指示を1回書くだけで、以降は短い指示で同じ品質のタスクが実行できる」点を強調。

#### Slide 12｜複合タスク ── ハンズオン（共通お題）

**タイトル：** 【ハンズオン・15分】複合タスクを実行してみよう

**Cowork へのタスク入力（コピーして使用）：**

```
デスクトップの「研修出力」フォルダにある「配送実績_元データ.csv」を使って、
以下の2つの成果物を同じフォルダに作成してください。

■ 成果物①：「配送実績_月次集計.xlsx」

- 上司向けの報告メール文面

### F. ソリューション設計＆次回予告（Slide 15〜18）

#### Slide 15｜Cowork ソリューション設計フローチャート

**タイトル：** 自分の業務課題にはどのパターンが向いている？

**フローチャート（大きく表示）：**

```
Q1. 解決したい業務の成果物は何か？
  ├─ 文書・メール・報告書 → 【文書生成タスク】
  ├─ Excel・集計表・グラフ → 【データ処理タスク】
  ├─ ファイルの整理・変換・リネーム → 【ファイル管理タスク】
  └─ 上記の組み合わせ → 【複合タスク】

Q2. その作業は定期的に繰り返すか？
  ├─ はい（毎日/毎週/毎月） → 【スケジュールタスク】に設定
  └─ いいえ（必要なときだけ） → 都度実行

Q3. 入力データはどこにあるか？
  ├─ ローカルファイル（Excel・CSV等） → フォルダを指定して読み込ませる
  ├─ Webから取得する情報 → Web検索と組み合わせる
  └─ 毎回手入力する → タスク指示文にテンプレートを含める
