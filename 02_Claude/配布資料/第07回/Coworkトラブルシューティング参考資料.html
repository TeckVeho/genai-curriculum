Cowork トラブルシューティング参考資料

利用対象： 主に講師用ですが、受講者がセルフで問題解決する際にも使えます。
第07〜11回のワークショップで手元に置いておいてください。

よくあるトラブルと対処法

トラブル①：Cowork タブが表示されない

症状： Claude Desktop を開いても「Cowork」タブが見当たらない。

対処法：

  1. Claude Desktop が最新バージョンか確認（設定 → アプリ情報）
  2. アプリを完全に終了して再起動する
  3. Pro 以上のプランでログインしているか確認（無料版では Cowork は利用不可）

トラブル②：タスクが完了しない / 途中で止まる

症状： タスクを送信してから長時間経っても完了しない。

対処法：

  1. 数分は待つ（複雑なタスクは処理に時間がかかる場合がある）
  2. タスクの途中で「状況を教えてください」とメッセージを送る
  3. 指示が複雑すぎる場合は、タスクを分割して小さく実行する
  4. Claude Desktop を再起動してやり直す

トラブル③：成果物ファイルが見つからない

症状： タスクは完了したが、指定フォルダにファイルがない。

対処法：

  1. Claude に「ファイルをどこに保存しましたか？」と聞く
  2. フォルダ名のスペルミスがないか確認する
  3. 出力先を明示的に指定し直す：「デスクトップの『研修出力』フォルダに保存してください」

トラブル④：Excel の関数・書式が期待と違う

症状： SUM関数の範囲が違う、色が指定と違う等。

対処法：

Excel の以下の点を修正してください：
- 〇〇セルの SUM 関数の範囲を B2:B10 に変更
- ヘッダー行の背景色を紺色（#003366）に変更
※ それ以外の部分は変更しないでください。

トラブル⑤：意図と違う変更をされた

症状： 「書式だけ変えて」と指示したら、データの内容まで変わってしまった。

対処法：

【変更してほしいこと】ヘッダー行の背景色を青にしてください。
【変更しないでほしいこと】データの内容・関数・その他の書式は一切変更しないでください。

コツ： 「変更してほしいこと」と「変更しないでほしいこと」を明示する

トラブル⑥：同じ指示で毎回違う結果が出る

症状： 同じタスクを実行しても出力の構成や文面が微妙に違う。

対処法：

  1. 指示文に出力の構造を明示する：「必ず①②③の順序で、以下のフォーマットで出力してください」
  2. プロジェクトの指示文に条件を入れると、毎回適用されて安定する
  3. 完全に同一の出力は保証されないが、構造・要素の指定で大幅に改善する

トラブル⑦：ファイルの削除確認が出て進めない

症状： Cowork がファイル削除の許可を求めてくるが、削除してよいかわからない。

対処法：

  1. 削除対象のファイル名を確認し、不要なものであれば「Allow」で許可
  2. 不安な場合は「削除しないでください。新しいファイルとして別名で保存してください」と指示

指示文のコツ（効果的な修正指示の書き方）

悪い例 良い例
「動きません」 「タスクが完了してもファイルが生成されません」
「間違っています」 「集計の合計行が B列だけ抜けています。B列にも SUM 関数を入れてください」
「直してください」 「〇〇の部分だけを修正してください。△△は変えないでください」
「もっとよくして」 「ヘッダー行の背景色を紺色、文字色を白にしてください」
