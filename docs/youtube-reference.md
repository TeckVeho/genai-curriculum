## 1. Codex とは

OpenAI の **ChatGPT の AI エージェント版サービス**。

| ツール | 役割 |
|---|---|
| ChatGPT | **考える** AI（文章生成・質問応答） |
| Codex | **実行する** AI（実際に作業を代行） |

- **週 400 万人弱**が利用（Claude Code と同水準）
- 2025年5月にリリース → 2026年2月にデスクトップアプリ公開 → 2026年4月のアップデートで一気に話題に

### 具体例

> 「先方への日程調整メールを作成し、そのまま Gmail で下書きまで用意して。送信は私の確認後でOK」

- **ChatGPT**：メール文面を生成して表示（送信は自分でやる）
- **Codex**：Gmail を開き、メールを作成し、**実際に送信まで完了**

## 2. ChatGPT との違い

| 項目 | ChatGPT | Codex |
|---|---|---|
| 主な用途 | 質問応答・文章生成 | タスクの実行 |
| 動作環境 | ブラウザ / アプリ | デスクトップアプリ / CLI / IDE |
| ファイル操作 | アップロードしたもののみ | **PC上の全ファイルを操作可能** |
| 外部ツール連携 | 限定的 | **90以上のプラグイン**、ほぼ何でも接続可 |
| 自動化 | 基本不可 | **スケジュール実行（オートメーション）可** |
| PC操作 | 不可 | **コンピューターユーズ機能あり** |
| 並列処理 | 不可 | **複数タスクの同時実行可** |

## 3. 基本的な使い方

### インストール

- **デスクトップアプリ**：OpenAI 公式サイトから Mac / Windows 版をダウンロード
- **CLI**：GitHub ページに記載のコマンドをターミナル（Mac）またはPowerShell（Windows）で実行

### 基本フロー

```
1. デスクトップアプリを起動
2. プロジェクト（フォルダ）を選択・作成
3. やりたいことを日本語で入力
4. Codex が自動で作業を実行
5. 結果を確認して承認
```

### プロジェクト（フォルダ）の考え方

- プロジェクトなしで作業すると**結果が保存されない**
- 「新しいプロジェクト → 最初から始める」でフォルダを作成
- 以降はそのフォルダ内で作業が完結する

### 権限設定

| モード | 内容 |
|---|---|
| デフォルト権限 | ちょっとした作業でも承認ポップアップが出る |
| 自動レビュー | AI が危険と判断したもの（ファイル削除など）だけ承認を求める |
| フルアクセス | 全て承認なしで進める |

> 用途に応じて使い分ける。迷ったらスクリーンショットを ChatGPT に貼って「これは何の承認を求めていますか？」と質問。

## 4. 内部モデル（GPT 5.5）

現時点で最新のモデルを使用（随時アップデートされる）。

| 項目 | 内容 |
|---|---|
| コンテキスト長 | **105万トークン**（日本語約80万文字・新書8冊分） |
| 最大出力 | **12.8万トークン**（長いコードや文章も途切れない） |
| マルチモーダル | 画像・音声・動画に対応 |
| 推論の深さ | 調整可能（高くするほど精度↑・時間↑） |

> 最新ベンチマーク確認は [LM Arena](https://lmarena.ai) を参照。

## 5. 主要機能

### 5-1. コンピューターユーズ（PC 操作）

Codex がマウスクリック・キーボード入力を自動実行し、**バックグラウンドで PC を操作**する機能。

**設定方法**
1. 左下の設定 →「コンピューターを使用」をオン
2. プラス → プラグイン → コンピューターを選択

**活用例**
- 指定した画像の削除
- デザインツールの操作
- Excel 作業の自動化
- 繰り返し単純作業の代行

> ※ ブラウザ操作と比べると**安定性が低く速度も遅い**ため注意。

---

### 5-2. プラグイン（外部アプリ連携）

外部アプリケーションと接続する機能。

**接続できる主なサービス**
- Gmail / Google Drive / Google Calendar
- Slack / Notion / Discord
- Microsoft Word / Excel
- その他 90 以上のプラグイン

**設定方法**：左上の「プラグイン」→ 対象サービスのプラスボタン → インストール

**活用例（Notion）**
> 「このマニュアルページに、テスト投稿のページを新しく追加してください」→ Notion に自動でページ作成

---

### 5-3. オートメーション（スケジュール実行）

決めた日時に自動でバックグラウンド実行する機能。

**活用例**
- 毎日のニュースまとめ
- 週次レポート自動作成
- 未対応タスクの抽出

**設定方法**：左上「オートメーション」→「チャットで作成」→ 自然言語でやりたいことと実行時間を伝える

> 例：「毎週日曜日14時にIT関連ニュースをURLつきでまとめてください」

---

### 5-4. 内蔵ブラウザ（Web 直接操作）

右上ボタンからサイドパネルを開くと**Codex 内でブラウザが使える**。コンピューターユーズより**安定・高速**。

**活用例**
> 「このページのお問い合わせフォームに移動して必要な項目を入力してください。名前は三浦ケイトです。2026年6月14日14時から打ち合わせをお願いしたいです」

→ ブラウザを自動操作してフォームに入力・送信

---

### 5-5. 画像生成（GPT Image 2）

**標準で GPT Image 2 が使える**ため、タスクの中に画像生成を含められる。

**活用例**
- 画像付き PowerPoint スライドの自動作成
- ゲーム用スプライトの生成 → そのままゲーム開発へ

---

### 5-6. メモリ機能

AI が好みや設定を覚えてくれる機能。

**設定**：左下の設定 → パーソナライズ → メモリをオン

**推奨運用**：全体メモリより **プロジェクト内に `AGENTS.md` を作成**する方が実用的。

```
理由：全体メモリだと余計な文脈が他の作業に混入してしまうため。
プロジェクトごとに AGENTS.md を用意することで、
そのフォルダでの作業時だけ読み込まれる。
```

### 5-7. その他の機能

| 機能 | 内容 |
|---|---|

## 6. 非エンジニアの活用例

### 例 1：毎朝の業界ニュース自動まとめ

オートメーション × Web 検索 × メールプラグイン

> 毎朝8時に業界ニュースを5件取得・要約 → 自分のメールアドレスに自動配信

### 例 2：未読メールの優先度リスト化

Gmail プラグイン × AI 分析

> 未読メールを確認し、返信が必要なものを優先度付きでリスト化 → Slack / LINE に通知

### 例 3：Excel データからプレゼン資料自動作成

Excel 読み込み × グラフ生成 × PowerPoint 作成

> Q1売上Excelを読み込み → データ分析 → グラフ付きスライドを自動作成 → Gmail で上司に送信

### 例 4：Slack の情報整理

Slack プラグイン × AI 要約 × オートメーション

> 先週の自分宛メンションと重要決定事項をまとめてレポート出力 → 週明けのキャッチアップが5分で完了

### 例 5：スケジュール管理の自動化

Google カレンダー × Google Drive プラグイン

> 今週のカレンダーを確認 → 各会議の関連資料をドライブから検索 → 準備リストとして整理

## 7. スキルで繰り返しを自動化

### スキルとは

**Codex 専用の手順書**。一度作れば何度でも呼び出せる。Claude Code でも同じ概念が使える（スキルを流用可能）。

- スラッシュコマンドで呼び出す
- 毎回同じ手順・品質で再現可能
- 繰り返し作業ほど効果大

### スキルの作り方（スキルクリエイター）

1. 左上「プラグイン」→ 右上「作成」→「スキルを作成」
   または `/` → スキルクリエイターを呼び出す
2. 作りたいスキルの内容を自然言語で説明（音声入力推奨）
3. AI が自動でスキルを生成

**例：比較画像生成スキルの作成**

> 「2つの要素を比較する画像を生成するスキルを作成してください。デザインは黒・水色・白のみ使用。テキストは黒か白のみ」

### 活用事例

| スキル名 | 内容 |
|---|---|
| マニュアル作成 | 動画を読み込み → 画像付き手順書を Notion に自動保存 |
| イベント用バナー | 説明文を入れるだけで、固定レイアウトのサムネ＋SNS投稿文をワンセット生成 |
| Substack バナー | タイトルをペーストするだけで、毎回同じデザインのバナー画像を生成 |

### スキルの共有・インストール方法

**他者からスキルを受け取った場合**

方法①（簡単）：スキルクリエイターを起動 → ファイルをドラッグ＆ドロップ →「このスキルをセットアップしてください」と指示

方法②（確実）：
```
Mac の場合：
Macintosh HD → ユーザー名 → 隠しファイルを表示（Cmd+Shift+.）
→ .codex → skills フォルダに直接配置 → アプリを再起動
```

## 8. Codex vs Claude Code 比較

| 項目 | Codex（OpenAI） | Claude Code（Anthropic） |
|---|---|---|
| 開発元 | OpenAI | Anthropic |
| モデル | GPT 系 | Claude 系 |
| 動作環境 | デスクトップ / CLI | デスクトップ / CLI |
| 設定ファイル | `AGENTS.md` | `CLAUDE.md` |
| 得意分野 | **実装力・画像生成** | **デザイン・Web UI** |
| 初心者向け機能 | **リモートコントロール** | **Co-work 機能** |
| API 料金 | **圧倒的に安い** | 高め（Max プラン 75ドル〜） |
| 利用制限 | 5時間あたり15〜350メッセージ | プランに応じた使用量上限 |
| CLI の使い心地 | 使いづらい | **使いやすい** |
| アプリの使い心地 | **使いやすい** | 普通 |

**使い分けポイント**
- **Codex が有利**：画像生成が必要、API コストを抑えたい、ChatGPT に慣れている
- **Claude Code が有利**：デザイン品質重視、CLI で使いたい

## 9. おすすめ機能

### 9-1. 音声入力

- 設定 → 一般設定 → 音声入力 → ショートカットキー登録
- Codex 内だけでなく**PC 全体**で使える音声入力になる
- 辞書登録で誤字脱字を削減

> ※ Super Whisper などと比べて「バックグラウンドにシステムプロンプトを埋め込む」機能はない

### 9-2. スマホによるリモートコントロール

スマホを Codex のリモコンとして使う機能。**Claude Code より使い勝手が良い**（スマホから新規セッションを作成できる）。

**設定方法**
1. 左下の設定 → 「接続」→「このMacを操作するデバイス」→ 追加
2. QR コードをスマホで読み取る
3. スマホの ChatGPT アプリ → サイドパネル → Codex から操作

**注意**：パソコンがスリープだと動かない → **常時起動の Mac mini などに設置するのが理想**

## 10. 最新アップデート

### AppShot

開いているアプリウィンドウを Codex のスレッドに瞬時に添付できる機能（スクリーンショットの手間を省く）。

- Mac：Cmd キー 2 回押し、またはプラスマーク → アプリ名を選択
- エンタープライズ以外の全プランで利用可能

### /goal（目標モード）

設定した目標が達成されるまで Codex が**自律的に作業を繰り返す**モード。

- デスクトップ版：プラスマーク →「目標を目指す」
- CLI 版：`/goal` コマンド

> 例：「このゲームを完成させて」→ 画像生成・スプライト化・BGM作成・コーディングまで自律実行

### 内蔵ブラウザ（ブラウザコード）

Codex 上のブラウザでページ要素を直接操作・指示できる機能（別途紹介済み）。

### プラグインの共有

自作プラグインを他者に配布できる機能。

> ※ 現状はビジネスプランのみ対応。個人プランでもファイルを直接渡す方法で共有可能。

## 12. まとめ・活用のポイント

### ChatGPT と Codex の使い分け

| 用途 | 推奨ツール |
|---|---|
| アイデア出し・文章作成 | **ChatGPT** |
| 定期的な作業の自動化 | **Codex** |
| 複数アプリの連携・管理 | **Codex** |
| 実作業の実行・ファイル操作 | **Codex** |

### AI エージェント活用の 5 つのメリット

1. **時間節約**：繰り返し作業の自動化で1日数時間を削減
2. **ミス削減**：サブエージェントが自動で検証・バグ修正
3. **ツール統合**：Excel・Gmail・Slack などを1画面で操作
4. **プログラミング不要**：自然言語だけで自動化できる
5. **スケーラブル**：複数タスクを並列処理して無限に拡張

### ワークフロー構築のステップ

```
1. プラグインで外部アプリを接続
2. スキルクリエイターで繰り返し作業をスキル化
3. オートメーションでスケジュール実行を設定
4. リモートコントロールで外出先からも管理
```

### プロンプトの基本構成

```
目的 + 文脈 + 制約 + 完了条件
```

> この4要素を意識するだけで成果が大きく変わる。
