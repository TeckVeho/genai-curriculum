## 1. オートメーション機能とは

**あらかじめ指定した時間・曜日に、設定したタスクを自動実行する機能**（スケジュール機能）。

### Codex が実行できる内容

| 種類 | 内容 |
|---|---|
| PC 上の作業 | ファイル作成・保存・操作 |
| アプリ連携 | MCP / プラグインで接続したアプリへのタスク実行 |
| 通知送信 | Slack / Discord などのメッセンジャーへプッシュ通知 |

### メリット

> 1度設定すれば、以降は自分が何も操作しなくても  
> **パソコン上にファイルが生成されている**、もしくは  
> **アプリ側にコンテンツが出来上がっている**  
> という状態を自動で作れる。

## 2. 設定方法

### 方法①：通常チャットから設定（かんたん）

通常の会話欄に「この自動化の設定をしてください」と自然言語で入力するだけ。

### 方法②：オートメーション画面から設定（確実）

1. 左上の「オートメーション」をクリック
2. 右上の「チャットで作成」をクリック
3. 表示されたデフォルトの文言に追加する形で指示を送る

> ②の方が確実に設定される傾向がある（デフォルトのシステムプロンプトが反映されるため）。

### 設定できる項目

| 項目 | 内容 |
|---|---|
| 繰り返しパターン | 毎日 / 毎週 / 特定曜日など |
| 実行時刻 | 任意の時刻を指定 |
| プロジェクト | どのフォルダで作業するかを選択 |
| モデル | GPT 5.5 など使用モデルを選択 |
| 推論レベル | 低（高速）〜高（高精度）で調整 |

### 作成後の管理

- 左上「オートメーション」一覧に追加される
- プロンプトは後から手動で編集可能
- 「今すぐ実行」ボタンでテスト実行できる

## 3. 事前準備（プラグイン設定）

オートメーションで**通知を受け取るアプリ**を事前に接続しておく必要がある。

### 推奨プラグイン：Slack

- スマホアプリがあるため、**プッシュ通知**としてホーム画面に届く
- チャンネルを用途別に分けて管理できる

**接続手順**
1. 左上「プラグイン」→ Slack を検索
2. インストール → Slack 側で ChatGPT アプリを連携

### その他の接続例

| 用途 | プラグイン |
|---|---|
| メール取得 | Gmail / Outlook |
| SNS・Web データ収集 | Bright Data |
| ファイル保存 | Google Drive |

## 4. 活用事例

### 活用事例① 毎日のニュース自動配信

**シナリオ**：毎朝 9 時に生成 AI 関連のニュースを Slack に通知する

**設定プロンプト例**
```
毎朝9時に生成AIに関するニュースをSlackにプッシュ通知してください。
この自動化の設定をしてください。
Slackのチャンネルは「ニュース」チャンネルに通知してください。
```

**結果**
- `ニュース` チャンネルに最新のAIニュースが自動配信される
- 以降は何も操作しなくても毎日届く

---

### 活用事例② 未対応メールの整理・通知

**シナリオ**：毎日 12 時に未返信メールを重要度付きで Slack に通知する

**事前準備**
- Gmail プラグインを接続

**設定プロンプト例**
```
アジア東京時間を基準に過去3日間に受信したメールを確認し、
未対応のメールを取得してください。

問い合わせの概要を伝え、以下の基準で重要度（高／中／低）を相対評価してください：
- 売上への影響
- 顧客影響度
- 期限
- クレーム度合い
- 返信の緊急性

結果をまとめた表をSlackの「重要メール」チャンネルに日本語で通知してください。
毎日12時に実行してください。
```

**結果**
- Slack の `重要メール` チャンネルに重要度付きのメール一覧が届く
- 返信忘れを AI が自律的に検知・通知してくれる

### 活用事例③ ウェブサイト更新監視

**シナリオ**：特定のウェブサイトが更新されたら Slack に通知する

**事前準備**
- **Bright Data プラグイン**を接続（スクレイピング専門サービス）
  - 世界中の Web サイトを合法・安全にスクレイピングできるサービス
  - 広告監視・競合サイト監視に活用可能

**設定プロンプト例**
```
https://example.com/news のページのニュースが更新されていたら、
SlackのWebサイト監視チャンネルに通知を送ってください。
更新されていなかった場合は通知不要です。
毎日朝9時に実行してください。
```

**動作の仕組み**

| 実行回 | 動作 |
|---|---|
| 初回 | 現状のページ内容を「基準データ」として記憶。通知なし |
| 2回目以降 | 前回との差分を比較し、更新があれば Slack に通知 |

**実証結果**
- テスト記事を追加 → 「テスト」ページが追加されたと Slack に通知が届いた（成功）

**応用例**
- 競合他社のサイト監視
- 採用ページの更新通知
- EC サイトの最安値アラート（Amazon・eBay など）

### 活用事例④ SNS バズコンテンツ収集

**シナリオ**：Instagram で AI 関連のバズっている投稿を自動収集して Slack に通知

**事前準備**
- Bright Data の MCP を使用

**設定プロンプト例**
```
Bright DataのMCPを使って、
Instagramで直近1週間にAIに関する発信で
フォロワー数に対して10倍以上バズっている投稿を
最大5つ見つけて、Slackのソーシャルチャンネルに送信してください。
```

**取得できるデータ**
- 投稿 URL
- フォロワー数
- 再生回数（リール動画の場合）
- いいね数・コメント数
- エンゲージメント倍率

**活用のポイント**
- **「フォロワー数が少ないのに再生数が多い投稿」** を自動抽出することで、バズりやすいネタを AI が発見してくれる
- リール動画を対象にする場合は「リール動画」と明示する
- 数字の組み合わせ方（倍率の定義）を具体的にプロンプトで指定すると精度が上がる

**その他応用**
- EC サイトの価格監視 → 最安値になったら通知
- 特定キーワードのトレンド監視

## 5. まとめ・活用のポイント

### オートメーション × プラグインの組み合わせ

```
① プラグインで外部サービスを接続（Gmail / Slack / Bright Data など）
② オートメーションでスケジュール実行を設定
③ 結果を Slack などに通知
→ 完全自動で情報が届く仕組みの完成
```

### 精度を上げるコツ

| ポイント | 詳細 |
|---|---|
| 出力形式を指定 | 「表形式で」「URLつきで」など具体的に |
| 対象を明確に | 「リール動画のみ」「過去3日間」など範囲を指定 |
| 評価基準を定義 | 重要度の判断基準（期限・売上影響など）を列挙 |
| 通知先を指定 | チャンネル名・言語まで明示 |

### ChatGPT との差別化ポイント

> これらの自動化は ChatGPT では**実現できない** Codex 独自の機能。  
> 設定は一度だけ。以降は何もしなくても情報が届く。

### 今後の発展

- より業務に特化したプロンプトへのカスタマイズ
- 複数のオートメーションを組み合わせたワークフロー構築
- 通知先の多様化（LINE / Discord / Google Chat など）
