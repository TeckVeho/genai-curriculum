# __第03回：Cowork入門 (1)：Coworkとは＋基本操作＋ドキュメント自動生成__

---

## __1\. 講義概要__

- __フェーズ:__ 座学
- __所要時間:__ 60分
- __主要機能:__ Claude Desktop — Cowork
- __形式:__ オンライン

### __狙い__

**Cowork** は「日本語でタスクを指示するだけで、Claude がファイル作成・データ処理・情報整理をまるごと自動実行してくれる」エージェント型ワークスペース。前回の Chat モード（プロンプト → 回答）から一歩進み、**「複雑な業務タスクを丸ごと任せて、完成品を受け取る」**という体験を通じて、第06回以降のワークショップで自分の課題を解決する力の土台を作る。

### __到達目標__

1. Cowork が「何ができる機能か」を自分の言葉で説明できる
2. Claude Desktop の Cowork モードに切り替え、タスクを1つ実行できる
3. 共通お題で文書ファイルの自動生成を体験し、成果物がローカルに保存される仕組みを理解している
4. 「自分の業務課題に Cowork をどう使えそうか」をイメージできる

### __準備物（講師）__

- プロジェクター・スクリーン（またはオンライン画面共有環境）
- 講師用PC（Claude Desktop をインストールし、Cowork が動作する状態にしておく。Pro 以上のアカウントが必要）
- 配布資料：**Cowork タスク設計メモ**（本テキスト4節参照）
- 事前に作成したデモ用タスク成果物（例：お詫びメールテキストファイル）

### __準備物（受講者）__

- PC（macOS または Windows）
- Claude Desktop アプリ（インストール済み。Cowork の実行には Pro 以上が必要。非対応の場合は講師アカウントで画面共有してペア体験）
- 前回の棚卸しシート（手元に置いておく）

### __【00\-10分】 Cowork とは何か__

- __フェーズ:__ 概念説明
- __スライド内容:__
	- __タイトル:__ Cowork ── 「やりたいこと」を伝えるだけで、Claude がまるごと実行する
	- __キーメッセージ:__ Chat は「1問1答」、Cowork は「仕事を丸ごとお任せ」。

- __講師スクリプト案:__  
「前回、プロンプトの4要素を学びました。Chat モードでは1つの質問に1つの回答が返ってきましたよね。  
**Cowork** はその発展形です。一言で言うと、**『複雑なタスクを日本語で依頼すると、Claude が計画を立て、ファイルを作り、データを処理し、完成品をあなたのPCに届けてくれる機能』**です。  

物流で例えると、Chat は『倉庫スタッフに1つずつ指示を出す』こと。Cowork は『仕事の目的を伝えて、やり方は任せる』こと。Claude が自分で計画を立て、必要なら複数の作業を並行して進め、最終的な成果物をファイルとして仕上げてくれます。」

- __Chat と Cowork の違い（スライド）:__

| | Chat モード | Cowork モード |
| --- | --- | --- |
| 操作場所 | ブラウザ（claude.ai）またはデスクトップアプリ | **Claude Desktop アプリ専用** |
| やり取り | 1問1答の会話 | **タスク単位**で丸ごと依頼 |
| ファイル操作 | アップロード → 回答で表示 | **ローカルファイルに直接読み書き** |
| 成果物 | 画面上のテキスト・Artifacts | **PC上のファイル（Excel・Word・テキスト等）** |
| 実行時間 | 数秒〜数十秒 | 数十秒〜数分（複雑なタスクはそれ以上） |
| 向いている作業 | 質問・相談・下書き | **ファイル作成・データ処理・整理・定期作業** |

### __【10\-20分】 デモ：Cowork を動かしてみる__

- __フェーズ:__ デモ
- __スライド内容:__
	- __タイトル:__ 実際に動かしてみましょう

- __講師スクリプト案:__  
「まず私が Cowork でタスクを実行するところを見てください。『納品遅延お詫びメール生成』というタスクです。」

- __デモ操作手順:__
	1. Claude Desktop を開き、上部のモード切替で **「Cowork」タブ** をクリック
	2. タスク入力欄に以下を入力して送信：  
	「取引先A社への納品遅延のお詫びメールを作成してください。遅延日数は2日、原因は台風の影響です。件名付き・丁寧なビジネス文体・250文字以内で作成し、デスクトップの『研修出力』フォルダに `お詫びメール_A社.txt` として保存してください。」
	3. Claude が計画を立て → 実行 → ファイルを書き出す過程を見せる
	4. デスクトップの「研修出力」フォルダを開き、ファイルが実際に作成されていることを確認
	5. 「B社・1週間遅延・倉庫トラブル」に変えて再度実行 → 毎回適切に変わることを確認

- __解説ポイント:__
	- 指示を出したら Claude が自動で計画を立てて実行すること
	- 成果物が**ローカルのファイル**として保存されること（アップロード・ダウンロード不要）
	- 進捗がリアルタイムで表示され、途中で介入・修正指示もできること

### __【20\-45分】 ハンズオン：自分で Cowork タスクを実行する__

- __フェーズ:__ 実践
- __スライド内容:__
	- __タイトル:__ 共通お題で Cowork を体験しよう（20分）

- __講師スクリプト案:__  
「では皆さんも実際にやってみましょう。共通お題はこちらです。」

- __共通お題:__  
**「取引先への納品遅延お詫びメールを Cowork で自動生成する」**  
入力：相手先名・遅延日数・遅延理由  
出力：件名付きのビジネスメール（250文字以内、丁寧な文体）がテキストファイルとして保存される

- __実行手順（スライドに手順書として提示）:__

	**Step 1：Cowork モードに切り替える**
	- Claude Desktop を開く → 上部の「Cowork」タブをクリック
	
	**Step 2：出力先フォルダを準備する**
	- デスクトップに「研修出力」フォルダを作成する（Claude にフォルダ作成を任せてもOK）
	
	**Step 3：タスクを入力して実行する**  
	以下をコピーして使用可：
	```
	取引先A社への納品遅延のお詫びメールを作成してください。

	条件：
	- 遅延日数：2日
	- 遅延理由：台風の影響
	- 件名を含める
	- 丁寧なビジネス文体（謙譲語を使う）
	- 250文字以内
	- 最後に「今後とも何卒よろしくお願いいたします」で締める
	
	デスクトップの「研修出力」フォルダに「お詫びメール_A社.txt」として保存してください。
	```
	
	**Step 4：実行結果を確認する**
	- Claude の実行過程（計画 → 作業 → 完了）を見守る
	- 完了後、「研修出力」フォルダを開いてファイルの中身を確認する
	
	**Step 5：修正指示を出してみる**
	- 「B社・1週間遅延・倉庫トラブル」に変えて、もう1通作ってみる
	- 出力が期待と違う場合は「もう少し丁寧に」「お詫びの部分をもっと強調して」等、追加指示を出す

- __巡回指導のポイント:__
	- 「Cowork タブが見つからない」→ Claude Desktop が最新版か確認。アプリを再起動
	- 「タスクが動かない」→ Pro 以上のプランか確認。無料版の場合は講師とペアで体験
	- 「ファイルが見つからない」→ 出力先フォルダのパスを確認。Claude に「ファイルをどこに保存しましたか？」と聞いてみる
	- 「指示文が思い浮かばない」→ 「普段どんなメールを書いていますか？その手順を言葉にするだけです」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「Cowork は無料版の Claude でも使えますか？」__
	- __A:__ 「Cowork の利用には **Claude Pro 以上** のプランが必要です（月額 $20）。無料版では Chat モードのみ利用可能です。Pro 契約がない方は、研修中は講師アカウントでペア体験していただきます。企業で Team / Enterprise プランを契約されている場合は全員が利用できます。」

- __Q2: 「Cowork の指示文は長い方がいいですか？」__
	- __A:__ 「必ずしも長い必要はありません。シンプルなタスクなら短くて明確な指示の方が安定します。出力が期待と違う場合は、足りていない条件を1つずつ追加していくのがコツです。」

- __Q3: 「Cowork で作ったファイルをやり直すことはできますか？」__
	- __A:__ 「もちろんです。実行後に『○○の部分を修正してください』と追加指示を出せます。また、最初からやり直すことも可能です。第06回以降のワークショップでは何度も修正を重ねて仕上げていきます。」

- __Q4: 「Cowork が作業中に意図しない変更をすることはありますか？」__
	- __A:__ 「Cowork はファイルを削除する前に必ず確認を求めます。また、実行過程がリアルタイムで表示されるので、おかしいと思ったら途中で止めることもできます。指示したフォルダ以外のファイルに影響を与えないよう、出力先を明示する習慣をつけましょう。」
