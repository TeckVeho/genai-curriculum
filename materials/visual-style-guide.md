# ビジュアルスタイルガイド — NotebookLM スライド生成用

> **目的:** 全回のスライドのトンマナ（トーン＆マナー）を統一するための視覚的デザインルール。  
> NotebookLM のソースとしてアップロードし、プロンプトから参照する。

---

## 1. 全体デザイン原則

- **ルック＆フィール:** クリーンでモダンなフラットデザイン。過度な装飾・グラデーション・3D表現は使わない
- **余白:** スライド四辺に十分なマージン（上下左右 5% 以上）を確保し、窮屈にしない
- **情報密度:** 1スライド1メッセージ。テキストは最大5〜6行。箇条書きは最大5項目まで
- **アイコン:** シンプルなライン（線画）スタイルで統一。塗りつぶしアイコンや写実的イラストは使わない
- **影・エレベーション:** カード要素には薄いドロップシャドウ（blur 8〜12px, opacity 10〜15%）を統一的に適用
- **角丸:** カード・ボタン・枠の角丸は 8〜12px で統一。完全な正方形や完全な円形カードは避ける

---

## 2. スライド型（テンプレート）定義

すべてのスライドは以下の12型のいずれかに分類する。各型のレイアウトルールを厳守すること。

### 型A: タイトルスライド

- **用途:** 各回の冒頭1枚目
- **レイアウト:** 中央揃え。大見出し（白 #FFFFFF, 42〜48pt Bold）を画面中央やや上に配置。サブテキスト（白 #FFFFFF, 20pt Regular）を大見出しの直下に配置
- **背景:** ネイビー（#02396B）の半透明オーバーレイ（opacity 75〜85%）を background.png の上に重ねる
- **装飾:** オレンジ（#F58220）の細い水平ライン（幅 60px, 高さ 3px）を大見出しとサブテキストの間に配置
- **右下:** 講師名・実施日を小さく表示（14pt, #FFFFFF, opacity 70%）

### 型B: カード横並び型

- **用途:** 振り返り、特徴紹介、ツール比較、分類ヒントなど
- **レイアウト:** タイトル行 + カード行の2段構成。カードは2〜5枚を等幅で横並び。カード間マージンは 16〜24px
- **カードスタイル:** 白背景（#FFFFFF）、角丸 10px、薄いドロップシャドウ。カード上部にメインカラー（#005BAC）の 4px ボーダートップ
- **カード内構成:** 上部にアイコンまたは番号（オレンジ #F58220 の丸、直径 36px、白文字）、中央にサブ見出し（Bold, 20pt）、下部に説明テキスト（Regular, 16pt）
- **3枚の場合:** 各カード幅 30%、間隔 3%
- **4〜5枚の場合:** 各カード幅を均等分割、テキストは簡潔に

### 型C: テーブル/比較表型

- **用途:** 機能比較、まとめ表、チェックリスト表、記入例
- **レイアウト:** タイトル行 + テーブルの2段構成。テーブルは左右マージン 10% でセンタリング
- **テーブルスタイル:** ヘッダー行はメインカラー（#005BAC）背景 + 白文字（Bold, 16pt）。データ行はストライプ（奇数行 #FFFFFF / 偶数行 #F0F4F8）。セル内テキスト（Regular, 15pt, #171717）。セル内パディング 12px
- **枠線:** 薄いグレー（#E0E0E0）の 1px ボーダー。外枠は角丸 8px
- **比較表の場合:** 左列（Before/旧）をグレーアウト、右列（After/新）を通常表示。強調列にはオレンジの左ボーダー（4px）を追加

### 型D: タイムライン/ステップ型

- **用途:** アジェンダ、操作手順、カリキュラムマップ
- **レイアウト:** タイトル行 + 縦タイムラインの2段構成。タイムラインは左 25% に時間/ステップ番号、右 75% に内容
- **ステップ番号:** メインカラー（#005BAC）の丸（直径 32px）に白文字の番号。丸同士を 2px のメインカラー線で接続
- **現在セクション/強調セクション:** オレンジ（#F58220）の丸 + オレンジの太線（3px）+ 背景にオレンジの薄い帯（opacity 8%）
- **テキスト:** 各ステップの見出し（Bold, 18pt）+ 補足（Regular, 15pt, #666666）

### 型E: Before/After型

- **用途:** 導入前後の比較、デモ結果の提示
- **レイアウト:** タイトル行 + 左右2カラムの2段構成。左カラム＝Before、右カラム＝After。中央に矢印アイコン（→）
- **Before 列:** 背景 #F5F5F5、テキスト #999999（薄いグレー）、上部に「Before」ラベル（グレー枠）
- **After 列:** 背景 #FFFFFF、テキスト #171717、上部に「After」ラベル（オレンジ #F58220 枠）
- **矢印:** 中央にオレンジの右矢印アイコン（32px）

### 型F: ワーク/タイマー型

- **用途:** ハンズオン作業中、個人ワーク中
- **レイアウト:** 画面中央に大きなタイマー表示エリア。下部 30% にガイドテキスト（ステップ番号付きリスト）
- **タイマー表示:** 大きなフォント（72pt Bold, #005BAC）で「⏱ ○○分」。メインカラーの薄い丸枠（border 3px, 直径 200px）で囲む
- **ガイドテキスト:** 番号付きリスト（Regular, 16pt）。現在ステップをオレンジで強調
- **背景:** 白基調。装飾は最小限

### 型G: チェックリスト型

- **用途:** 到達目標、確認ポイント、まとめ
- **レイアウト:** タイトル行 + チェックリストの2段構成。リストは左マージン 15% から開始
- **チェックボックス:** メインカラー（#005BAC）の角丸四角（20px）。未チェック＝空白枠線、チェック済み＝塗りつぶし+白チェックマーク
- **リスト項目:** 各項目は Regular 18pt、行間 1.8。項目間に 12px の余白
- **強調項目:** オレンジ（#F58220）のチェックボックス + Bold テキスト

### 型H: Q&A型

- **用途:** よくある質問コーナー
- **レイアウト:** タイトル行 + Q&Aカード（縦並び、1スライド2問まで）
- **Q カード:** 左端にメインカラー（#005BAC）の「Q」バッジ（Bold, 20pt, 白背景に青文字、丸枠）。質問テキスト（Bold, 18pt）
- **A カード:** 左端にオレンジ（#F58220）の「A」バッジ（同スタイル）。回答テキスト（Regular, 16pt）。背景にベースカラー（#F5F5F5）の帯
- **Q/A ペア間:** 16px の余白 + 薄いグレー（#E0E0E0）の水平区切り線

### 型I: フローチャート型

- **用途:** 判断フロー、プロセスフロー
- **レイアウト:** タイトル行 + フローチャートの2段構成
- **判断ノード（ひし形/角丸四角）:** メインカラー（#005BAC）の枠線（2px）+ 白背景。テキスト（Bold, 16pt）
- **結果ノード:** 角丸四角。Skills＝メインカラー（#005BAC）背景+白文字、Codex＝オレンジ（#F58220）背景+白文字
- **矢印:** グレー（#999999）の 2px 線 + 矢印。分岐ラベル（「はい」「いいえ」）を矢印上に表示（Regular, 14pt）

### 型J: アンケート/QR型

- **用途:** フィードバックアンケート、URL案内
- **レイアウト:** タイトル行 + 中央にQRコード/URLエリア + 下部に補足テキスト
- **QRエリア:** 画面中央に 240×240px 相当のスペース。白背景の角丸カード（16px）にドロップシャドウ
- **補足テキスト:** 「全○問・約○分」などを Regular 18pt で中央揃え

### 型K: コンテンツスライド型

- **用途:** 概念説明、機能紹介、問いかけ、デモシナリオなど汎用
- **レイアウト:** タイトル行 + 本文エリアの2段構成。本文エリアは左マージン 10%、右マージン 10%
- **強調メッセージ:** オレンジ（#F58220）の左ボーダー（4px）付きカード。背景 #FFF8F0（オレンジの極薄）
- **問いかけ文:** メインカラー（#005BAC）の大きなフォント（Bold, 28pt）で中央配置
- **箇条書き:** メインカラーの丸ドット（8px）。テキスト Regular 18pt、行間 1.8

### 型L: 配布資料型

- **用途:** 印刷配布物（シート、テンプレート、まとめ表）
- **レイアウト:** 白背景（#FFFFFF）。ヘッダーにタイトル（Bold, 24pt, #171717）。本文エリアにテーブルまたは記入欄
- **テーブル:** 黒枠（#333333, 1px）。ヘッダー行は薄いグレー（#F0F0F0）背景。セル内パディング 8px
- **記入欄:** 点線の下線（#CCCCCC, 1px dashed）。十分な行間（2.0）で手書き記入スペースを確保
- **装飾:** 最小限。背景画像は使用しない。印刷時に読みやすいコントラストを優先

## 3. 共通コンポーネントルール

### プログレスバー（固定ヘッダー）

- **位置:** スライド上部、全幅
- **高さ:** 40px
- **背景色:** サブカラー（#02396B）
- **テキスト:** 左側にセクション名（Regular, 14pt, #FFFFFF）、右側にスライド番号「○/○」（Regular, 14pt, #FFFFFF, opacity 70%）
- **進捗インジケーター:** メインカラー（#005BAC）の水平バーで進捗割合を示す

### 橋渡しメッセージ（つなぎメッセージ）

- **位置:** スライド下部 10%
- **スタイル:** イタリック風（Regular, 16pt, #666666）。左に薄いグレーの縦線（2px, #E0E0E0）を添える

### 絵文字の使い方

- **使用可:** カード型のラベル部分、テーブルの装飾列のみ
- **使用不可:** 見出し、本文テキスト、プログレスバー
- **スタイル:** 絵文字の代わりにラインアイコンを優先。絵文字を使う場合は1スライド最大3〜4個まで

---

## 4. リファレンス画像の使い方

NotebookLM のソースに、以下のリファレンス画像を追加してください（`materials/reference-slides/` に格納）:

| ファイル名 | 対応するスライド型 |
|---|---|
| `ref-title.png` | 型A: タイトルスライド |
| `ref-cards.png` | 型B: カード横並び型 |
| `ref-table.png` | 型C: テーブル/比較表型 |
| `ref-timeline.png` | 型D: タイムライン/ステップ型 |
| `ref-timer.png` | 型F: ワーク/タイマー型 |

上記のリファレンス画像のレイアウトとビジュアルスタイルを踏襲してください。
