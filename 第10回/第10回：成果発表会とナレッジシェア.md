# __第10回：成果発表会とナレッジシェア__

---

## __1\. 講義概要__

- __フェーズ:__ 成果
- __所要時間:__ 60分
- __主要機能:__ —（全機能の統合発表）
- __形式:__ オフライン

### __狙い__

作成した AI ソリューション（カスタムGPT / Skills / プロジェクト・トリセツ）を全員で共有し、**「あの人のノウハウ」を「みんなの標準装備」にする**。発表では **Before/After の数字・使った ChatGPT 機能の選定理由** も共有し、研修後も自律的に機能を選べる力を定着させる。終了時に **事後アンケート** を実施し、企業様には **Before/After 比較レポート** を後日提出する。

### __到達目標__

1. 自分が作ったソリューションを2〜3分で発表し、Before/After と機能選定理由を説明できる
2. 他者のソリューションを理解し、自分の業務にも応用できるか考察できる
3. チームの共有ライブラリ（デジタル・ツールセンター）にソリューションを登録できる
4. 事後アンケートに回答し、研修の Before/After を振り返ることができる

### __準備物（講師）__

- 各自の成果物（カスタムGPT URL / Skills 設定 / プロジェクト設定・トリセツ・設計図・機能選定理由メモ）
- 共有フォルダへのアクセス権限（社内サーバー、Teams、Google Drive など）
- 事後アンケート用 Google フォームの URL（または紙）
- 拍手をする雰囲気づくり

### __準備物（受講者）__

- 発表用成果物（カスタムGPT 名 or URL / Skills 名 / プロジェクト名）
- トリセツ（第09回で作成した3行マニュアル）
- Before/After メモ（業務時間の変化）
- 機能選定の理由メモ

### __【08\-35分】 リレープレゼン & 実演（Before/After/Demo）__

- __フェーズ:__ 発表
- __スライド内容:__
	- __タイトル:__ 改善事例リレー（お一人様 2〜3分）
	- __発表フォーマット（4点）:__
		1. **積荷（課題）：** 何が大変だった？（例：毎月の請求書チェック）
		2. **機能選定の理由：** なぜその ChatGPT 機能を選んだ？（例：繰り返す文書作成 → カスタムGPT、複数ステップの丸投げ → エージェントモード）
		3. **ソリューション（デモ）：** どんな AI を作った？実際に動かして見せる
		4. **成果（着時間）：** どれくらい早くなった？（例：1日20分短縮）

- __講師スクリプト案:__  
「では、リレー形式で発表をお願いします。堅苦しいプレゼンは不要です！ 朝の申し送りだと思って、シンプルに話してください。  

発表の例をお見せします：  
**Before：**『取引先からのPDF請求書を手で一覧に転記するのに毎回15分かかっていました。』  
**機能選定：**『画像入力が得意な ChatGPT に渡して表抽出するカスタムGPT を作りました。繰り返しの作業なのでカスタムGPT が最適でした。エージェントモードも試しましたが、この業務は1ステップで完結するのでシンプルなカスタムGPT の方が合っていました。』  
**デモ：**（実際に画像をアップロードして出力を見せる）  
**After：**『確認と修正だけで3分になりました。月末の残業が1時間減りました。』  

……こんな感じでOKです。では一番手の方、お願いします！」

- __演出:__
	- 発表が終わるたびに、率先して大きな拍手を送る
	- 「その機能選定の理由は聞けてよかった！」「これは総務部全員に使ってほしいですね」等のポジティブなコメントを入れる
	- 「セカンドオピニオンで別機能も試した方は、ぜひ比較した感想も一言どうぞ」と促す

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「他の人のカスタムGPT を使ってみて、うまくいかなかったらどうすれば？」__
	- __A（回答）:__ 「作った人に『ここどうやるの？』と気軽に聞いてください。教わったコツをトリセツに追記（赤入れ）してあげると、ツールがどんどん使いやすくなります。**Instruction や Few-Shot の追記は5分でできます。** 小さな改善を積み重ねることが大切です。」

- __Q2: 「ChatGPT がアップデートされて、今まで使えていたカスタムGPT が動かなくなったら？」__
	- __A（回答）:__ 「道路工事でいつもの道が通れなくなるのと同じです。焦らず Instruction を微修正してください。多くの場合、禁止事項や出力形式の指示を少し具体化するだけで動くようになります。困ったら講師または社内の IT 推進担当に相談してください。」

- __Q3: 「自分が作ったカスタムGPT、大したことない気がして共有するのが恥ずかしいです。」__
	- __A（回答）:__ 「とんでもないです！ あなたにとっての『当たり前』は、他の人にとっての『目から鱗』です。『全角を半角に直すだけ』のカスタムGPT でも、毎日100回入力する人にとっては神ツールになります。あなたが作ったものは必ず誰かの役に立ちます。自信を持って入庫してください。」

- __Q4: 「研修が終わったら、困ったときはどこに相談すればいいですか？」__
	- __A（回答）:__ 「まず**デジタル・ツールセンター**のトリセツを確認してください。それでも解決しない場合は、今日の参加者同士のチャット（Slack / Google Chat 等）に質問を投稿してください。仲間からの回答が集まります。また、研修後も定期的なフォローアップセッションを検討します（詳細は後日ご案内します）。」

### __登録された成果物の使い方ルール__

1. **Read Me：** 必ずトリセツ（3行マニュアル）を読んでから使うこと
2. **Kaizen：** 使ってみて「もっとこうしたい」と思ったら、作成者にフィードバックする
3. **Update：** 自分が改善したら、バージョン番号（v1.1 など）を上げて上書き保存する
4. **Security：** 個人情報・機密情報は入れない（全機能共通）。別紙『AI入力OK_NGチェックリスト』と社内ポリシーを守ること

---

### __研修で学んだ ChatGPT 機能まとめ__

| 機能 | 向いている業務 |
| --- | --- |
| テキスト生成（プロンプト4要素） | メール・文書・日報の下書き |
| Web 検索 | 最新情報の簡易リサーチ |
| 画像入力（ビジョン） | 手書き文字起こし・書類翻訳 |

| コードインタープリター | データ集計・グラフ化 |
