# __第08回：[ワークショップ] ソリューション構築 (2)：改善・機能追加__

---

## __1\. 講義概要__

- __フェーズ:__ 実践
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第07回で作ったプロトタイプを**実用レベルに引き上げる**。成果物の品質向上（書式・構成の改善）、機能追加（グラフ・集計・複数ファイル出力等）、エッジケース対応（入力データが不完全な場合等）を行う。「段階的に指示を重ねてソリューションを育てる」感覚を定着させる。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. 前回の「改善したいことリスト」を2つ以上反映できる
2. 「うまくいかなかった指示」を「うまくいく指示」に修正する経験を積める
3. 次回（仕上げ）に向けて残り作業を整理できる
4. 「自分の業務で実際に使えそうか」を具体的にイメージできる

### __準備物（講師）__

- PC（Claude Desktop の Cowork が動作する状態）
- 前回の各受講者の進捗メモ（第07回の終わりに確認した内容）

### __準備物（受講者）__

- PC（Claude Desktop が利用可能な状態）
- 第07回で作ったプロトタイプ（成果物ファイル・指示文が保存済みの状態）
- 第07回テイクアウェイの「改善したいことリスト」

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

---

### __【00\-08分】 前回の振り返り & 本日のゴール確認__

- __スライド内容:__
	- __タイトル:__ 今日のゴール：プロトタイプを「使える状態」に育てる

- __講師スクリプト案:__  
「前回はプロトタイプを作りました。今日はそれを**実際の業務で使えるレベル**に仕上げます。  
まず全員に一言ずつ確認します。前回の終わりに書いた『改善したいことリスト』を見せてください。今日はそのリストを潰していくのが仕事です。」

- 講師が全員の進捗を確認（2〜3分）。完成できていない受講者は今日の前半で追いつく
- 前回完成できていない受講者向け：「最初の15分は前回の続きを進めてください。完成したら改善に入ります」

### __【08\-50分】 改善・機能追加ワーク__

- __フェーズ:__ 個人作業（講師は全員を巡回）

---

#### __改善メニュー（例）__

以下のメニューから「改善したいことリスト」に合わせて選択し、Cowork に修正指示を出す。

| カテゴリ | 改善内容の例 | Cowork への指示例 |
| --- | --- | --- |
| **書式改善** | Excelの見た目を整える | 「ヘッダー行を紺色背景・白文字にしてください。金額列に3桁カンマ区切りを設定してください」 |
| **機能追加** | グラフを追加する | 「集計シートに配送先別金額の棒グラフを追加してください」 |
| **機能追加** | 複数ファイル出力 | 「集計Excelに加えて、上司向けサマリーメール文面をtxtファイルとして出力してください」 |
| **機能追加** | PowerPoint出力 | 「集計結果を3枚のスライドにまとめたPowerPointファイルも作ってください」 |
| **品質安定** | 出力の構成がばらつく | 「必ず以下の構成で出力してください：①タイトル ②集計表 ③グラフ ④コメント」 |
| **エッジケース** | データが不完全な場合 | 「金額が空欄の行がある場合は『要確認』と注記して、集計からは除外してください」 |
| **定期タスク化** | スケジュール設定 | `/schedule` コマンドで毎月1日に自動実行する設定を追加 |
| **プロジェクト改善** | 指示を恒久化 | プロジェクトの指示文に改善内容を反映し、次回以降も自動適用されるようにする |

---

#### __作業の進め方__

1. 改善したいことを1つ選んで修正指示を作成
2. Cowork に投入 → 成果物を確認
3. 期待通りなら次の改善項目に移る / 違えば追加の修正指示を出す
4. 時間まで繰り返す

- __巡回時の指導ポイント:__
	- 「1つ直したら必ず成果物を確認してから次へ。複数を一度に変えると何が効いたかわからなくなります」
	- 「Cowork が意図と違う変更をした場合は『〇〇は変えないでください。△△だけ修正してください』と限定的に指示を出してください」
	- 「行き詰まったら『現在の成果物の問題点を教えてください』と Claude に聞いてみましょう」
	- 「プロジェクトの指示文を更新すると、次回以降のタスクにも自動的に反映されます」

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「改善したいことが多すぎて、何から手をつけるかわかりません」__
	- __A:__ 「まず『使えない理由になるもの』から直してください。例：成果物が生成されない・内容が全然違う。書式の改善は後回しでOKです。」

- __Q2: 「同じ指示を出しても毎回違う結果が返ることがあります」__
	- __A:__ 「AIの特性上、完全に同じ出力を保証することは難しいです。ただし指示が具体的であるほど安定します。出力の構造（必ず①②③の順で）や条件（文字数・形式）を明示すると改善します。プロジェクトの指示に入れると毎回適用されるので効果的です。」

- __Q3: 「成果物のファイルが大きくなりすぎます」__
	- __A:__ 「出力の範囲を限定する指示を追加してください。例：『直近3ヶ月のデータのみ集計してください』『サマリーは300文字以内にしてください』。データ量が多い場合は、入力ファイル自体を絞り込んでから渡すのも有効です。」
