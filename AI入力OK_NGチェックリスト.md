# AI入力 OK / NG チェックリスト（研修配布用）

物流事務向け「生成AI活用・業務効率化研修」で、**何をAIに入力してよいか**を迷ったときの判断用です。不明なときは**上司または情報管理担当に確認**してから入力してください。

---

## 1. 判断フロー（3ステップ）

```
Step1: その情報は「誰でもネットで見られる公開情報」か？
  → YES なら Step2 へ
  → NO なら 原則 NG（企業版ライセンスと社内ルールの両方を確認してから）

Step2: 特定の個人・取引先・自社を識別できる内容が含まれるか？
  → YES なら 伏せ字・サンプル化してから OK の可能性（下記「グレーゾーン」参照）
  → NO なら 多くの場合 OK

Step3: 利用しているのは「無料の個人向けGemini」か、「会社契約の企業向け（Workspace / Advanced 等）」か？
  → 無料版: 機密・社内文書は入れないのが安全
  → 企業版（契約で学習に使われない等が明記されている）: 社内ルールに沿えば社内文書も可
```

---

## 2. OK（入力してよい例）

| 例 | 補足 |
| --- | --- |
| 業界一般の質問（「請求書の一般的な記載項目は？」等） | 特定企業に紐づかない内容 |
| **公開されている**法令・ガイドライン・標準約款の**抜粋や要約依頼** | 出典が公開情報であること |
| 自社名・取引先名を **A社・B商事** のように伏せた業務フロー説明 | 金額・個人名も伏せる |
| 研修用の**ダミーデータ**・**架空の数値** | 「これは架空です」と明記して生成させてもよい |
| すでに社外に出して問題ない広報文案・一般論のメールの骨子 | 社内承認済みの範囲で |

---

## 3. NG（入力しない）

| 例 | 理由 |
| --- | --- |
| **顧客・取引先の個人名、電話番号、メール、住所** | 個人情報・特定リスク |
| **社内の運賃表、掛率、未公表の契約条件** | 営業秘密 |
| **未発表の売上・利益・人事** | インサイダー・機密 |
| **従業員のマイナンバー、口座、健康情報** | 個人情報保護 |
| **パスワード、APIキー、認証コード** | セキュリティ事故の原因 |

※「ボカしたから大丈夫」は**不十分**なことがあります。中身が推測できる場合は NG 扱いにしてください。

---

## 4. グレーゾーン（社内ルールとライセンスで決める）

| 内容 | 無料版 Gemini（個人） | 会社契約の企業向けAI（例: Workspace の契約範囲内） |
| --- | --- | --- |
| 社内マニュアル・就業規則・経理規程の全文 | 原則 **避ける** | 社内ポリシーで **許可されている場合のみ** |
| 議事録・メールのやりとり | 固有名詞を除いた要約方針のみ、など **最小限** | 許可範囲内なら可。ソース管理に注意 |
| 取引先からの請求書PDF | **個人特定・金額が入るなら NG** | マスキングまたは社内手順に従う |

**鉄則:** 迷ったら **入れない**。上司・情シス・コンプライアンスに確認。

---

## 5. 入力前のワンチェック（コピー用）

- [ ] 個人を特定できる情報はないか  
- [ ] 自社の機密・未公開の数字はないか  
- [ ] パスワードや鍵はないか  
- [ ] 会社の「AI利用ポリシー」に照らしてよいか  
- [ ] 出力は **必ず人間が検品** する前提で使う
