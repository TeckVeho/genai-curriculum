# __第09回：[ワークショップ] ソリューション構築 (3)：仕上げ・テスト・プロジェクト化__

---

## __1\. 講義概要__

- __フェーズ:__ 実践（最終仕上げ）
- __所要時間:__ 60分
- __形式:__ オフライン

### __狙い__

第07〜08回で構築したソリューションを**発表できる状態に仕上げる**。最終調整・テスト・プロジェクト化（指示・ファイル・メモリをまとめて永続化）・スケジュールタスク設定（該当者のみ）を完了させる。また発表会（第10回）に向けた**デモ手順・説明資料**を準備する。終了時に**簡易アンケート**を実施する。

### __到達目標__

1. ソリューションを「誰かに引き継げる状態（プロジェクト化済み・テスト済み）」に完成させられる
2. 3行運用マニュアル（使い方のメモ）を作れる
3. 第10回の発表に向けて Before/After・デモ手順を準備できる
4. 自分が作ったソリューションの限界・今後の改善点を言語化できる

### __準備物（講師）__

- PC（Claude Desktop の Cowork が動作する状態）
- 第10回の発表フォーマット（本テキスト4節参照）

### __準備物（受講者）__

- PC（Claude Desktop が利用可能な状態）
- 第07〜08回で作ったソリューション（成果物・指示文・プロジェクトが保存済みの状態）
- 第08回テイクアウェイの「次回タスクリスト」「発表会準備メモ」

- __講師スクリプト案:__

---

### __【05\-40分】 仕上げ作業__

- __フェーズ:__ 個人作業（講師は全員を巡回）

---

#### __仕上げ作業リスト（優先順に取り組む）__

**① 残機能の完成（第08回タスクリストの残り）**  
未完了の改善項目を Cowork への指示で対応する。

**② 動作テスト（必須）**  
以下のテストシナリオで動作確認を行う。
- [ ] 通常の入力データ・条件で期待通りの成果物が出力されるか
- [ ] 入力データが不完全・異常な場合に適切に対処できるか（エラー or 注記）
- [ ] 成果物の書式・構成・文体が設計書の仕様に沿っているか
- [ ] ファイルが指定フォルダに正しく保存されるか

**③ プロジェクト化（推奨）**  
繰り返し使うタスクの場合、Cowork のプロジェクトとしてまとめる：
1. プロジェクト名・指示文が設定されているか確認
2. 必要なファイル（テンプレート・マスターデータ等）がプロジェクトに追加されているか確認
3. プロジェクト内でタスクを再実行し、指示文だけで期待通りの成果物が出力されるか最終確認

**④ スケジュールタスク設定（該当者のみ）**  
定期実行が必要な場合：
1. `/schedule` コマンドでスケジュールを設定
2. 実行頻度（毎日/毎週/毎月）を確認
3. サイドバーの「Scheduled」で設定内容を確認

**⑤ 3行運用マニュアルの作成**  
Cowork に以下を指示して作成：

> 「私が作った Cowork タスク『[タスク名]』の使い方を、PC操作が苦手な事務担当者向けに3ステップで説明する文章を書いてください。各ステップは1文で、小学生にもわかる言葉で書いてください。デスクトップの『研修出力』フォルダに `運用マニュアル.txt` として保存してください。」

### __【40\-50分】 発表準備（全員共通）__

- __フェーズ:__ 発表準備
- __スライド内容:__
	- __タイトル:__ 第10回の発表準備をしよう

- __講師スクリプト案:__  
「残り20分で発表準備をします。第10回は1人あたり**2〜3分**の発表です。以下の順番で話してください。」

- __発表フォーマット（スライドで提示・配布）:__

	1. **業務課題（Before）：**  
	「私は毎週〇〇という業務に△△時間かかっていました」
	
	2. **作ったソリューション：**  
	「Cowork で〇〇タスク（/ プロジェクト）を作って解決しました」
	
	3. **デモ（30〜60秒）：**  
	実際に Cowork でタスクを実行し、成果物が生成される様子を見せる
	
	4. **After（見込み）：**  
	「これを使うと、週〇〇時間 → 〇〇時間に短縮できる見込みです」
	
	5. **今後の展望・感想：**  
	「次はこんな改善をしたい」「研修で特に役立ったこと」を一言

- __発表準備ワーク（10分）:__
	- 発表フォーマットに沿ってメモを作成
	- デモ手順を1回通しで確認する（失敗しても大丈夫、練習なので）

## __3\. 想定される質問と回答 \(Q&A\)__

- __Q1: 「ソリューションがまだ完成していません。第10回までに間に合いますか？」__
	- __A:__ 「第10回の冒頭に10分の最終作業時間を設けます。ただし完成度より『何を作ったか・何を解決しようとしたか』の方が大切です。動かない部分があっても、課題・設計の意図・今後の方向性を発表できれば十分です。」

- __Q2: 「発表が苦手で緊張します」__
	- __A:__ 「発表会は競争ではありません。同じ受講者同士、お互いの業務を知り合う場です。『ここがうまくできた』も『ここで詰まった』も、両方話してOKです。」

- __Q3: 「作ったソリューションを研修後も使い続けることはできますか？」__
	- __A:__ 「はい。Claude Desktop の Cowork は、Pro 以上のプランが有効な限り使い続けられます。プロジェクト化したソリューションは指示・ファイル・メモリがまとめて保存されているので、そのまま実務で運用できます。改善したい場合は追加指示を出すだけです。」
