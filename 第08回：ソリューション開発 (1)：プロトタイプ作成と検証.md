# __第08回：ソリューション開発 (1)：Gems作成とカスタムAI構築__

## __1\. 講義概要__

- __狙い:__ 第07回の設計図をもとに、**Gems（カスタムAI）**として「教育済みの自分専用チャット」を1つ完成させる。完璧を目指さず、**動くたたき台**と**隣席での交換テスト**までを60分で行う。
- __準備物:__
	- ノートPC（Google アカウントログイン済み）
	- __Gemini Advanced 等、Gems が使える環境__（使えない場合は「Instruction を冒頭に付けたマイプロンプト」をメモ帳に保存するフォールバックで代替）
	- __第07回の「AI業務設計フォーマット」__（記入済み）
	- __宿題共有用チャット（Slack / Google Chat 等）の案内__

## __2\. 講義スクリプト & スライド構成 \(60分詳細\)__

### __【00\-5分】 宿題シェアタイム ＋ ゴール設定__

- __フェーズ:__ 振り返り / 導入
- __スライド内容:__
	- __タイトル:__ 今日は「仮組み」の日＝Gems 1本完成
	- __キーメッセージ:__ NotebookLM の新規構築は第09回。今日は **Gems に集中**。
- __講師スクリプト案:  
__「皆さん、お疲れ様です！ 最初の5分、前回の宿題（設計図をGeminiに投げてみた結果）をチャットか口頭で共有してください。  
今日のゴールは一つ。**第07回の設計図から、Gems（自分専用AI）を1本作る**ことです。  
『開発』と聞くと身構えますが、新人さんに業務マニュアルを渡して横乗りする感覚でOKです。**完璧な積載率は求めません。まずパレットに載せる**ことが大事です。」

### __【05\-20分】 Gems の仕組み解説 ＋ 講師デモ__

- __フェーズ:__ デモ・講義
- __スライド内容:__
	1. __タイトル:__ あなた専属の「事務アシスタント」を作ろう（Gemini Gems）
	2. __キーメッセージ:__ 毎回同じ前置きは不要。「いつもの型」を Gem に埋め込む。
	3. __ビジュアルイメージ:__ 名札をつけた複数のキャラ（「請求確認係」「社内連絡係」など）。
- __講師スクリプト案:  
__「Gems は、Gemini に**役割・口調・禁止事項**をあらかじめ教えた『専属チャット』です。  
例：**『取引先向け・請求に関する連絡メール作成係』**。Instruction に『結論から、です・ます調、時候の挨拶なし、金額は必ず確認欄を設ける』と書いておく。  
呼び出したら、あとは**『再発行依頼／支払延期のお願い』**など用件だけ入れれば、毎回そろった形の下書きが出ます。」
- __デモ/操作手順:__
	1. Gemini の **Gem マネージャー**（Gems 管理）を開く。
	2. __名前:__ 「取引先向け・請求連絡メール係」
	3. __Instruction（指示）例:__

```
あなたは物流会社の一般事務担当です。
ユーザーから「用件」「相手の属性（既存取引先／新規）」「希望日」を受け取ったら、丁寧で簡潔なビジネスメールの本文案を作成してください。
・です・ます調、結論から
・時候の挨拶は書かない
・金額や個別契約に関する断定は避け、「要確認」と明記する
・文末は業界慣習に合わせ「ご安全に」等の一言を入れてよい
```

	4. 保存し、「用件：請求書の再発行依頼、希望：今週中、相手：既存取引先A社（社名は伏せ字）」と入力してテスト。

__■ Gems が使えない場合のフォールバック__

- 上記 Instruction と、毎回の用件を**1つのプロンプトに結合**し、メモ帳に「マイプロンプト」として保存。Gemini の通常チャットにコピペする（第03回のフォールバックと同じ考え方）。

### __【20\-45分】 開発タイム：自分の設計図から Gem を作成__

- __フェーズ:__ 演習（個人作業）
- __スライド内容:__
	- __タイトル:__ 設計図の Input / Process / Output を Instruction に落とし込む
	- __キーメッセージ:__ 「AIに何をさせるか」をそのまま英語・日本語で書いてよい。まず動かす。
- __講師スクリプト案:  
__「お手元の設計フォーマットの③④⑤を見てください。**Process と制約**を、Gem の Instruction に書き写します。**Output の形式**（箇条書き、表、メールなど）も Instruction に明記してください。  
20分以上かかってもいいので、**最低1本、保存まで**やり切ってください。困ったら手を挙げてください。巡回します。」

### __【45\-55分】 交換テスト：隣の人が使えるか__

- __フェーズ:__ 演習（ペア）
- __スライド内容:__
	- __タイトル:__ 「他人が使える」がチーム展開の第一歩
	- __キーメッセージ:__ 用件だけ渡して、意図通りか確認。
- __講師スクリプト案:  
__「隣の人とペアになり、**自分の Gem に相手の業務っぽい用件**を1つだけ入れてもらってください。  
『意味がわからない出力』『足りない条件』があれば、Instruction を1行足す。**第09回で Few-Shot（見本）を足して精度を上げます。**」

### __【55\-60分】 まとめ ＋ 次回予告__

- __フェーズ:__ まとめ
- __スライド内容:__
	- __タイトル:__ 今日の積荷：Gems 1本＋改善メモ
	- __キーメッセージ:__ 次回は **NotebookLM 構築** と **Few-Shot・マニュアル・車検**。
- __講師スクリプト案:  
__「お疲れ様でした！ 今日は **Gems だけ**に集中しました。次回は **NotebookLM** で規程・マニュアルを載せたノートを作り、今日の Gem とあわせて **見本（Few-Shot）** を足し、**簡易マニュアル**まで仕上げます。  
宿題：**今日作った Gem（またはマイプロンプト）を、今週中に業務で1回だけ使う**。結果をメモして第09回の冒頭で共有してください。」

## __3\. 想定される質問と回答 \(Q&A\)__

### __Q1\. 「設定するのに時間がかかって、自分で打った方が早い気がするんですが…」__

A\. 「1回だけなら手打ちの方が早いです。でも年間100回の業務なら、今日の20分は投資です。**一番繰り返しが多い業務1つ**に絞ってください。」

### __Q2\. 「Gems の回答が時々おかしいです。」__

A\. 「**助手の検品**が必要です。第09回で Few-Shot と制約を足します。ハルシネーションに注意し、機密は『__AI入力OK_NGチェックリスト__』に従ってください。」

### __Q3\. 「手書きのメモは Gem で読ませますか？」__

A\. 「画像を貼れるなら可能ですが、今日の主役は**定型の文章・表の生成**です。画像読み取りは第03回のマルチモーダルと組み合わせて設計してください。」

## __4\. 本日のTakeaway \(配布資料用\)__

__【お持ち帰りシート：Gems（カスタムAI）の極意】__
